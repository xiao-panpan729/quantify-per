# 缠论结构定位层接入

> 2026-06-01 决策记录

## 背景

用户从 2007 年开始研究缠论，持续约 10+ 年，后放弃。但重新评估后认为缠论的分型→笔→中枢框架提供了笔记本系统缺失的 **标准化结构描述语言**——"当前价格在走势的哪个位置"。

## 项目选择

**czsc (waditu/czsc)** — GitHub 上最流行的缠论 Python 实现。

| 维度 | 评估 |
|------|------|
| 版本 | v0.10.12 (2026-03-08)，仍在活跃更新 |
| 架构 | Rust + Python 混合，核心算法编译为 .pyd |

| 文档 | 作者明确表示暂不写文档，需源码阅读 |

## 实现分析：czsc vs 缠论原文

核心差异：
- **分型**: ✅ 完全符合原文（中间K线最大/最小 + low条件）
- **包含处理**: ⚠️ 方向自适应，不继承前一笔方向
- **成笔**: 笔终点选极值而非"第一个"，但有动态固化机制（被破坏则删除合并）
- **线段**: ❌ 缺失。作者立场："没了分型笔线段还是缠论吗？如果答案是'是'，这就是为你准备的"
- **中枢**: 直接用笔构建（跳过线段），30分钟及以上级别可用
- **买卖点**: 有完整的一/二/三类买卖点信号函数，但基于笔形态模式匹配，非严格递归定义

**最有价值特性**: 多级别联立分析（CzscTrader 原生支持日线+30分钟同时分析）

## 适配方案

不复制 czsc 源码（43k行），保持 pip 依赖，写适配层：

```
tools/chanlun/
├── adapter.py    — DataFrame → czsc RawBar
├── signals.py    — CZSC → 一/二/三买卖点
├── positions.py  — 日线+30分钟双级别联立定位
```

## 协作关系

```
czsc 提供：     结构定位（分型→笔→中枢→买卖点区域）
用户系统叠加：   ★买/★卖闭环 + 信号质量评分 + CCI验证
最终：          缠论说位置 + 系统说时机 = 综合决策
```

**Why：** 缠论买卖点是区域概念（底部背驰区/回调确认区），不是精确进场点。用户系统的信号系统正好在这个区域里做精细化判断。

**How to apply：** 笔记本系统的技能应同时利用缠论结构信息（"当前在日线上涨笔+30分钟买点区域"）和现有指标（★买/CCI/评分）做综合预测。
