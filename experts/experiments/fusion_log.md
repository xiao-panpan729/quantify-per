## 实验 #1 — 势能+红线+叙事 首轮融合 (2026-06-11)

### 背景
专家系统框架未定型，但势能筛选(6号)和叙事(7号)已有可用输出。用 `redline_breakout_screener.py` 做首次融合测试——验证"技术指标选股+叙事等级过滤"这条链路是否走得通。

### 参与专家
| 专家 | 角色 | 输出 |
|------|------|------|
| 势能筛选(6号) | 动量筛选 | x₁ 强度 ≥ 8 |
| 红线突破 | 技术形态过滤 | 突破牛熊红线(MA221+3σ)、压制比、突破天数 |
| 叙事(7号) | 质量验证 | 叙事链 S/A/B/C/U 等级 |

### 运行配置
```python
势能阈值: ≥ 8
压制比: ≥ 0.8
突破天数: ≤ 3天
甜点区: 势能8-11, 突破1-2天, 压制>85%
叙事等级: S/A/B/C 纳入, U标记待研报精读
```

### 迭代过程

**Round 1 (15:22)** — 默认参数初筛 → 44只
- 势能≥8, 压制比≥0.8, 突破≤60天

**Round 2 (15:26)** — 加板块标注 → 77只
- 带了sector标签重新输出

**Round 3** — 收紧突破窗口 → 32只
- 突破天数从60→3天，剔除大部分老突破

**Round 4** — 甜点区过滤 → 3只
- 势能8-11, 突破1-2天, 压制>85%
- 中捷精工(11.7)、莱伯泰科(12.0)、百合花(12.7)

**Round 5** — 叙事交叉验证
- 当时映射桥把"特斯拉→新能源车(S)"
- 中捷精工因为有"特斯拉"板块标签 → 显示S级

**Round 6 (16:33)** — 叙事映射修正
- 用户指出：中捷精工是冲压件小厂，给特斯拉供座椅骨架，年利润几千万，没分析师覆盖
- 发现映射桥粒度过粗，同板块内核心/边缘股无区分

### 关键发现

1. **映射桥不能"板块→链"一刀切** — 同一个通达信板块里的股票叙事质地可以天差地别。特斯拉板块下有宁德时代也有中捷精工，不能都归同一个S级
2. **引入U级（未覆盖）** — 没有研报精读验证的品牌级板块，不强行塞入已有体系。U级正确位置
3. **品牌链正规流程确立** — 写时间线→写模板→更新判定层→再改映射。不能跳级
4. **甜点区参数初步有效** — 势能8-11+突破1-2天+压制>85% 筛出的3只结构上确实干净
5. **纯技术筛选不够** — 没有叙事验证，中捷精工/莱伯泰科/百合花在量价上都很漂亮，但叙事实地完全不同

### 后续行动（本日完成）
| 行动 | 状态 |
|------|------|
| 5品牌链并行研报精读（特斯拉/苹果/小米/华为/英伟达） | ✅ |
| 特斯拉机器人 #51 B级 时间线+模板+判定层 | ✅ |
| 苹果AI终端 #52 A级 时间线+模板+判定层 | ✅ |
| 小米汽车 #53 A级 时间线+模板+判定层 | ✅ |
| 华为/英伟达 → 不设独立链，子主题归入已有体系 | ✅ |
| 中捷精工 S级→小米汽车 A级 修正 | ✅ |
| 映射桥重建，U级加入 | ✅ |

### 结论
势能+红线+叙事 三者结合的选股链路走得通，但叙事映射桥的质量是整个管线的瓶颈。没有可靠的叙事等级输入，技术指标选出的股可能全是"假突破+假故事"。

## 实验 #2 — 信号适配器层落地：StandardSignal + 双适配器 (2026-06-15)

### 背景
专家融合框架设计定型后进入实施阶段。第一步是统一输出协议——定义 StandardSignal 数据结构，写适配器翻译专家原生输出，不改动专家内部代码。

### 参与专家
| 专家 | 适配器 | 原生输出 → StandardSignal |
|------|--------|--------------------------|
| 宏观板块(2号) | adapter_macro_sector.py | macro_sensitivity.json(环境分类→C) + sector_momentum_cache.json(板块x₁→Pow) |
| 量领(5号) | adapter_volume_leader.py | 信号CSV(buy_signal/sell_signal→S/G, MACD状态→ABCD级别, 量能确认→Pow) |

### 标准信号结构体
```python
StandardSignal = {
    "expert_id": str,       # 专家唯一ID
    "stock_code": str,      # 标的代码
    "timestamp": int,       # Unix时间戳
    "signal_type": str,     # cycle/event/report
    "S": int,               # 方向：-1/0/1
    "C": float,             # 置信度：0.0~1.0
    "Pow": float,           # 强度：0.0~1.0
    "G": str,               # 操作级别：A/B/C/D或空
    "raw_data": dict        # 原生数据，溯源用
}
```
实现路径：`experts/signal_adapter/standard_signal.py`

### 适配器运行结果

**宏观板块适配器**（14只跟踪标的）：
- 环境分类：宽松 → C=0.70
- 有板块映射个股6只（通威/TCL/中环等），Pow 0.12~0.40
- ETF/指数8只无板块映射，Pow=0（按设计，ETF走板块层级）

**量领适配器**（80只量领宇宙）：
- ★买：41条  |  ★卖：36条
- 级别分布：A=74(96%)，B=3(4%)，C/D=0
- 多数信号带量能确认（vol_突放/vol_梯度升加成 Pow=0.60~0.80）

### 联合查询演示
同一标的两个适配器联合输出 → 简易融合：
```
sh600487 CPO/光通信  宏观:C=0.70 P=0.18  量领:min5买 C=0.90 P=0.80 G=A  → 看多 C=0.63 P=0.61
sh603986 存储芯片    宏观:C=0.70 P=0.34  量领:min15买 C=0.90 P=0.75 G=A → 看多 C=0.63 P=0.63
sh600176 PCB/新材料  宏观:C=0.70 P=0.38  量领:min30卖 C=0.90 P=0.50 G=A → 看空 C=0.63 P=0.46
```

### 关键发现
1. **适配器可行** — 不改动专家内部代码即可将异构输出翻译为统一格式，16行数的适配器/专家，翻译出1条 StandardSignal
2. **量领级别集中度极高** — 96% A级，说明量领宇宙天然筛选了强势股。等级分化要到弱势环境才会出现
3. **宏观乘数起统一修正作用** — C_multi = C_vl × 0.70（环境宽松），用同一个乘数修正所有信号，简单有效
4. **ETF覆盖缺口是设计问题** — ETF无板块映射是正常的，按信号层级体系（§9）ETF走板块层覆盖
5. **信号层级体系确认** — 不是所有标的一个层级：ETF≈板块层，个股是终端层

### 待解决问题
- 势能选股适配器（x₁是连续排名，S/C/Pow映射方式待定）
- 融合引擎骨架（级别路由+置信过滤+共振计算）
- 现有`demo_joint_query.py`展示的简易融合（乘法+加权）可作为融合引擎雏形

## 实验 #3 — 缠论适配器落地 + S/Pow/C/G 纯净映射 (2026-06-15)

### 背景

实验#2落地了 StandardSignal + 量领/宏观双适配器。本次补齐第三个适配器：**缠论（ta_chanlun）**。缠论的结构化输出天然适合翻译为 S/C/Pow/G，但方向逻辑经过三轮迭代才定案。

### 参与专家

| 专家 | 适配器 | 状态 |
|------|--------|------|
| 技术分析-缠论(1号) | adapter_chanlun.py (248行) | **本次新增** |
| 技术分析-量价(0号) | — | 未启动 |
| 宏观板块(2号) | adapter_macro_sector.py | ✓ 实验#2 |
| 量领(5号) | adapter_volume_leader.py | ✓ 实验#2 |

### 缠论映射规则

```
S   ← 纯结构（线段方向 / 中枢突破 / 中枢偏向），背驰不参与
Pow ← 背驰衰减率 = (进入段-离开段) / 进入段；无背驰用结构力度
C   ← 背驰类型(0.30~0.80) + 突破确认 + 共振加成 + 买卖点确认
G   ← A(trend) / B(consolidation) / C(minor)
```

### 设计迭代（三轮）

| 轮次 | S逻辑 | 问题 |
|------|-------|------|
| v1 | 五因子加权求和→反推S/C/Pow | 维度混合，值不可解释 |
| v2 | 中枢震荡+背驰→S跟背驰方向 | sh588200在涨却给S=-，用户:"背驰不是空的理由" |
| **v3** | **S=纯结构，背驰只影响Pow/C** | ✅ 当前版本 |

**关键语义修正**：背驰≠方向翻转。顶背驰+上涨线段≠S=-，只表示力度衰减→可能转震荡。方向由结构决定，力度由背驰决定，二者独立。

### 底层修复：GG/DD 中枢扩张检测

`notebook/chanlun/positions.py` `_check_level_div` 合并中枢时只检查 ZG/ZD，忽略 GG/DD。导致 sz300118 的 4 个中枢（GG/DD全部交叉穿透）被误判为独立趋势中枢→A级趋势背驰。

修复：新增 GG/DD 交叉检查 → 中枢扩张自动合并 → sz300118 从 A→B 级。
详见 `notebook/research_log.md` 实验#26。

### 运行结果（14只标的）

- 12条 B级盘整背驰 + 2条无背驰
- 无 A 级（GG/DD修复后虚假趋势背驰全部消失）
- 多8条 / 空6条

### 待解决

- 剩余5个适配器：势能选股/基本面/信息聚合/叙事/研报
- 融合引擎骨架（级别路由+置信过滤+共振计算）
- `__init__.py` 中 adapter_volume_leader 的 bare import 在包导入时会失败

### 结论

缠论适配器验证了"不改专家内部、只加翻译层"的路线适用于结构化专家。最重要的设计教训：**背驰是力度信号不是方向信号**，这个语义区分决定了整个映射框架的正确性。

### 迭代2：S规则修正 + 信号分段（2026-06-15 下午）

用户审查全量输出后指出三个问题：

**问题1 — 新高+顶背驰/新低+底背驰 不应有方向**
之前S=纯结构方向，但用户提出：价格创新高+顶背驰（力度衰减）时方向不明确。
修复：顶背驰+S=1→S=0，底背驰+S=-1→S=0。

**问题2 — 中枢震荡偏强/偏弱需配合大级别趋势**
之前偏强→S=+1、偏弱→S=-1。用户指出偏强/偏弱只是中枢内位置，方向要看日线趋势。
修复：偏强+日线向上→S=1，偏强+日线向下→S=0（逆势反弹）；偏弱同理。

**问题3 — 价格单位不匹配导致假突破（positions.py bug）**
`_zhongshu_segment` 把CSV原始close（sh000001: 409）传给CZSC归一化的中枢边界（GG=0.43）→ 409≥0.43 → 假"突破中枢上轨"。修复：用 `daily_bars[-1].close` 替代原始CSV值。

**信号分段概念引入**
历史事件（二买/三买）≠ 当前信号。label中历史事件用括号包裹：
- before: "中枢震荡，下行笔已终结，反弹中，30分钟二买信号"
- after:  "中枢震荡，下行笔已终结，反弹中（30分钟二买信号）"

**运行结果变化**
- sh000001: 假"突破中枢上轨"→正确"中枢震荡，中枢内部"，S=+→S=0
- 全市场S分布：S=+仅1只、S=-仅1只、S=0共12只（真实反映中枢震荡市）
