**定位**: 专家系统是量化系统的顶层框架。9个独立专家 → 统一输出协议 → 上层路由调度，替代原有预测卡路线。
**模式**: 单文件持续更新，做到一定程度后再考虑拆分。对标 `filter_evolution.md`（量领）和 `notebook/research_log.md`（笔记/DL）。

> 笔记项目（notebook/）是专家系统下的一个研究支撑模块，不是与专家系统平级的框架。

## 一、专家分类体系

按功能域分为三类专家，各有独立编号：

```
                      ┌──────────────────────────┐
                      │    路由调度层（未来待建）    │
                      │   场景识别 → 专家选择/组合  │
                      └────────────┬─────────────┘
                                   │
        ┌──────────────────────────┼──────────────────────────┐
        │                          │                          │
   ┌────▼─────────┐        ┌──────▼────────┐        ┌───────▼────────┐
   │  底层分析专家  │        │   选股专家     │        │   支撑专家      │

   │   基本面/事件) │        │   红线组合)    │        │   信源)        │
   └───────────────┘        └───────────────┘        └───────────────┘
```

### 关键关系

- **选股专家（5-6号）** 消费 **底层分析专家（1-4号）** 的指标 + **叙事专家（7号）** 的等级标签，综合输出选股清单
- **叙事专家** 是基础设施，不直接输出交易信号，为选股提供"故事质量"维度
- **选股专家之间**并列不互斥：量领侧重成交额广度，势能筛选侧重动量强度，各自跑各自的结果
- **风控专家** 硬拦截：任一路由结果触发风控 → 否决

### 执行顺序

**第一步：统一专家输出（当前阶段）** — 每个专家输出 `{direction, conviction, horizon, evidence[], risks[]}`
**第二步：再建路由层** — 场景识别器 + 门控网络 + 决策融合

### 关键约束
- 不拆现有模块，只加协议层
- 每个专家可独立迭代、独立下线
- 选股专家之间不互斥，各自出结果后由上层做综合
- 预测卡路线已冻结

### 2.1.1 缠论3K线原语分类理论（2026-06-23 小草×小潘潘联合推导）

本节的推导路径：单根K线12种形态 → 三根组合的数学答案C(14,3)=364 → 缠论语义层面≈7类。**结论：364是数学答案，7类是交易答案。**

---

#### 第一步：单根K线有12种

K线OCHL四个价格的相对关系，在H≥O/C且L≤O/C约束下，枚举归一化后共12种：

| 类别 | 种数 | 明细 |
|:--|:--|:--|
| 特殊线 | 4 | 一字线 / T字 / 倒T字 / 十字星 |
| 阳线 | 4 | 无影 / 上影 / 下影 / 上下影 |
| 阴线 | 4 | 无影 / 上影 / 下影 / 上下影 |

#### 第二步：三根组合的数学答案

从12种中取3根，可重复、不计顺序（多重集合组合）：

- **无序**（不区分排列顺序）：C(12+3-1, 3) = C(14, 3) = **364**
- **有序**（区分K线先后顺序）：12³ = **1,728**（数学真，交易假）

**364是纯数学答案。** 真实交易中大多数组合不具备语义意义。

#### 第三步：缠论语义分类（≈7大类）

从缠论视角，3根K线关心的维度屈指可数：
- **分型判断**：H₂是否最高 / L₂是否最低
- **包含关系**：B₂与B₁/B₃的区间是否重叠
- **方向感知**：C₁→C₂→C₃是否构成连续方向
- **转折试探**：涨跌涨 / 跌涨跌的特殊形态

对应7大类（用 sz159740 最近500根日线验证）：

| 大类 | 占比 | 判定逻辑 |
|:--|:--|:--|
| **① 方向序列** | 26.9% | C₁<C₂<C₃（简单上涨1016%）或 C₁>C₂>C₃（简单下跌16.3%） |
| **② 包含关系** | 22.9% | B₂与B₁/B₃区间重叠。其中61%方向不明需看前文趋势，23%向上处理，12%向下处理 |
| **③ 顶分型** | 16.9% | H₂ > H₁ 且 H₂ > H₃ |
| **④ 底分型** | 15.3% | L₂ < L₁ 且 L₂ < L₃ |
| **⑤ 转折试探** | 13.3% | 涨跌涨/跌涨跌，其中涨跌涨中的吸收4.0%>假突破1.6%，跌涨跌中的诱多4.4%>反转2.6% |
| **⑥ 其他/复杂** | 4.8% | 不满足上述任一条件 |
| **合计** | 100% | |

#### 关键结论

- **364 ≠ 交易语义**。数学组合数不回答交易问题
- **缠论7类 ≈ 3K线的信息极限**。再多就是噪声
- **包含关系的方向依赖前文趋势**（14.1%的窗口无法自足判定），这是缠论"合并K线前先看趋势"原则的数据验证
- **涨跌涨/跌涨涨 ≈ 15%**，和顶/底分型同一量级——它们是"未完成的分型"或"分型的失败变体"

#### 第四步：Token框架（走势的"词元"模型）

**核心思想**: 3根K线组合 ≈ 语言模型中的Token（词元）。单根K线像"字母"没有语义，3根K线的7类组合才是带有语义的最小单位。

**Token切分方式**: 步进=1的滑动窗口
```
K线:    1   2   3   4   5   6  ...
Token:  A(1,2,3) → B(2,3,4) → C(3,4,5) → D(4,5,6) ...
```
- 每生成一根新K线，滑动窗口前移一格，生成一个新Token
- Token数量 = K线数量 - 2（与K线数一一对应）
- 没有"空格"：走势没有自然边界，重叠滑动是合理的取义

不与NLP单词类比（单词有自然边界），而与**语音识别声学特征帧**类比（帧与帧重叠，连续不中断）。

**与之前工作的关系**:
- 2026-06-18 Kronos-mini 失败的原因：裸OHLCV输入 → 模型在学"字母序列"而非"单词序列"，4.1M参数不足以从零推导缠论结构
- 新方案：**Token序列模型**。输入从连续OHLCV改为离散Token类型（7分类），问题从回归（预测价格）降为分类（预测下一个Token），复杂度降一个量级
- 之前Phase 1.0标注的24,851行训练数据（笔方向/中枢/分型/趋势状态）可作为辅助标注层，与新Token框架互补

**关键设计决策**:

2. **MAD去极值而非3σ**: 财务数据分布厚尾严重，3σ会剔除过多有效样本（如高增长公司的真实极端值），MAD(中位数绝对偏差)更鲁棒
3. **TYPE A/B 双分类**: A类=净利润持续高增长的PEG叙事（机构抱团风格），B类=营收扩张→CAPEX投入→折旧高峰→FCF转折的重资产生命周期。两类不是排他的，一个公司可以同时触发A和B的条件
4. **因子溢价 vs 因子暴露分离**: 溢价（factor premium）是市场给这个因子的奖励程度（时间序列平均），暴露（factor exposure）是单个股票在这个因子上的得分。两个维度正交，分开输出

1. **关键词子串匹配（非LLM分类）**: 用正则关键字匹配板块名称→叙事等级，速度快（毫秒级）、结果稳定可重复。避免LLM分类的不确定性
2. **品牌产业链拆分子链**: 华为概念（1000+股）不笼统定级，拆分为华为AI服务器(A)、华为昇腾(A)、华为汽车(B)等子链各给等级
3. **S/A/B/C/U五级制**: S=超级主线（AI/半导体/新能源），A=强叙事（人形机器人/商业航天），B=有故事（可控核聚变），C=边缘（禽流感），U=未分类。U级不强行塞入S/A/B/C，标记待研报精读
4. **外资观点聚合**: 单独维护 foreign_views/ 目录，每条观点带原文链接和时间戳。内外资分歧点是重要的市场拐点信号

### 2.8 风控专家（8号）— 支撑

**核心定位**: 硬拦截模块。触发即否决——不产生信号，只否决危险信号。回答"当前能不能做？"

**模块清单**:

| 组件 | 来源 | 职责 |
|------|------|------|
| ABCD级别 | cycle_engine/grading.py | 日线MACD状态→最小操作周期筛选 |
| 两轴决策 | cycle_engine/grading.py + engine.py | 纵轴(ABCD)管周期可信度，横轴(zone_advice)管该不该做 |
| fragile_high检测 | cycle_engine/engine.py | 11+分虚高警示，评分高但结构脆弱时禁止追高 |
| D级拦截 | cycle_engine/grading.py | D级(日线MACD<0或死叉)时禁止任何做多操作 |

**任务目标**:
1. 防止在错误的时间周期入场（ABCD周期筛选）
2. 防止在虚高区域追高（fragile_high 11+分禁止买入）
3. 防止在弱势环境下频繁交易（D级+中性区不参与）
4. 两轴独立决策，不互相替代——大盘好也不能在D级重仓

**使用说明**:
```
# 风控信号自动集成在 cycle_report.json 和 synthesized_report.json 中
python run_cycle.py --save                 # 更新所有标的的风控等级
python -c "import json;d=json.load(open('signals/tracking/_signals/cycle_report.json'));[print(f'{i[\"name\"]}: {i[\"grade\"]} {i[\"zone_advice\"]}') for i in d]"
```

**当前能力评估**:
- ✅ ABCD级别 + 两轴决策框架成熟可用
- ✅ fragile_high 虚高警示成功案例（多次阻止高位追入）
- ✅ D级硬拦截逻辑清晰（MACD<0或死叉时不参与）
- ⚠️ 风控逻辑散落在grading.py和engine.py中，没有独立的风控模块入口
- ⚠️ 没有统一的"风控报告"输出——目前需要去cycle_report.json里捞
- ⚠️ 量价背离、情绪极端值等额外风控维度尚未接入

**关键设计决策**:
1. **两轴独立（不替代原则）**: 纵轴ABCD管"哪个周期可信"，横轴zone_advice管"该不该做"。大盘趋势好(横轴偏多)不能在D级(纵轴偏弱)重仓——两轴不互相替代
2. **fragile_high反转逻辑**: 趋势评分11+但评分结构脆弱（闭环薄弱）→虚高警示。这不是信号错误，而是"评分正确但结构预示反转"——是unique的卖出信号
3. **ABCD→最小操作周期**: A级=5分钟信号可入场，B级=需5分钟★买+2次金叉，C级=需15分钟★买+2次金叉，D级=不参与。越弱势越需要大级别确认

## 三、统一输出协议（v1 设计中）

### 标准 Schema

```
{
  expert:     "expert_name",
  stock:      { code, name },
  timestamp,

  verdict: {
    direction:  "bullish" | "bearish" | "neutral",
    conviction: 0.0 ~ 1.0,
    horizon:    "short" | "mid" | "long"
  },

  evidence: [
    { key: "factor_name", value: "factor_value", weight: 0.0~1.0 }
  ],

  risks: [
    "判断失效条件描述"
  ]
}
```

### 叙事专家 conviction 计算

| 因子 | 权重 | 映射 |
|------|------|------|
| 叙事等级 | 0.4 | S→0.95 / A→0.80 / B→0.55 / C→0.30 / U→0.10 |
| 链覆盖数 | 0.2 | ≥3条→↑ / 1条→↓ |
| 等级变化方向 | 0.2 | 升级→↑ / 稳定→ / 降级→↓ |
| 生命周期位置 | 0.2 | 新兴→↑ / 共识→ / 过热→↓ / 衰退→↓ |

direction: S/A→bullish, B→neutral, C/U→bearish
horizon: mid/long（叙事天然偏中长期）

### 势能专家 conviction 计算

| 因子 | 权重 | 映射 |
|------|------|------|
| x₁ 当前位置 | 0.4 | >8(过热→↓) / 3-8(强势→↑) / <2(弱势→↓) |
| x₁ 百分位 | 0.3 | >80%过热 / 20-80%正常 / <20%超卖 |
| x₁ 趋势(5日) | 0.2 | 加速→↑ / 减速→↓ / 反转→↓ |
| 量能配合 | 0.1 | 放量+高x₁→↑ / 缩量+高x₁→↓ |

direction: x₁>5→bullish, 1-5→neutral, <1→bearish
horizon: short（势能天然偏短期）

## 八、关系图谱 —— 日报地基工程

**触发**: 2026-06-16，用户发现 gen_daily_brief 每次重新生成的话题选择不一致。

**根因**: 日报的"重要性判断"没有结构化底层支撑。LLM 看到一堆平铺的文章分组，没有被告诉"在你的投资逻辑体系里，什么更重要"。结果每次独立 LLM 调用像重新掷骰子。

### 8.1 当前现状：散件齐备，未拼装

| 组件 | 数据 | 回答什么问题 |
|------|------|------------|
| `node_map.json` | 270板块 × 3750节点 | 什么板块在什么时候大涨过 |
| `macro_history.json` | 3068个A/B级节点宏观标注 | 当时发生了什么宏观事件 |
| `tdx_sector_narrative_map.json` | 概念板块 → S/A/B/C/D 叙事等级 | 每个板块对应哪条产业链叙事 |
| `star_buy_node_bayes.json` | ★买信号 → 节点 → 贝叶斯分组 | 历史上这类节点有多大胜率 |
| `annotate_node_events.py` | 1176个节点 / 5043事件 | 每次行情背后有什么产业政策事件 |

**关键缺失**: 这些组件各自产出了结构化数据，但从来没有人把它们拼成一张可查询的图。没有图，日报引擎不知道：
- "美伊协议"在传导链上是什么位置（一级驱动变量？二级传导？）
- 下游哪个板块/个股是"当前最亮"的节点
- 一个事件是增量信息还是已经 fully priced in

### 8.2 目标：构建可查询的关系图谱

```
事件（美伊协议）
  │
  ├─ [变量层级] 核心变量 → 定价锚（油价/通胀/Fed路径）
  │   └─ [传导链] 油价↓ → 通胀预期↓ → 加息概率↓ → 科技股估值修复
  │       └─ [点亮节点] 英伟达链(S级) / 半导体产业链 / AI算力链
  │       └─ [点亮标的] NVDA / AMD / 中际旭创...
  │
  ├─ [变量层级] 结构性变量 → 产业逻辑（光通信技术路线）
  │   └─ [传导链] CPO推迟 → NPO加速 → MPO/FAU受益
  │       └─ [点亮节点] 光通信 / PCB上游
  │       └─ [点亮标的] 易中天...
  │
  ├─ [变量层级] 结构熊市 → 结果而非原因
  │   └─ 不点亮节点（这是下游结果，不是可交易变量）
  │
  └─ [变量层级] 情绪噪音 → 无传导链
      └─ 不点亮（无结构性影响）
```

### 8.3 重要性判断的五维框架

日报需要的不再是关键词计数，而是五维打分：

| 维度 | 权重 | 判定逻辑 | 当前状态 |
|------|------|---------|---------|
| **变量层级** | 30% | 核心变量(定价锚) > 结构性变量(产业逻辑) > 下游结果 > 情绪噪音 | 需新建 |
| **传导链位置** | 25% | 越接近上游（油价/利率/地缘），影响面越大 | 部分有（transmission_chains.md） |
| **定价状态** | 20% | 边际变化（新增量）vs 已被定价（延续） | 需新建 |
| **节点亮度** | 15% | 点亮了多少S/A级叙事→板块→标的 | 需从散件拼装 |

## 九、时序模因图谱架构 —— 系统终极架构蓝图

**触发**: 2026-06-18 凌晨，用户与豆包深度对话 Transformer K线预测 → 多模态融合 → 知识图谱完整推导。
**命名**: 用户自创，六字概括三层架构。

### 9.1 核心结构（三层叠加）

```
        上层：图谱层（资金流 + 产业关系网络）
            ↑ 强化方向筛选
        中层：模因层（方法论：缠论/★买/六步法/量领等）
            ↑ 方向确认 + 信号增强
   ┌────────┴────────┐
 底层A：K线预测大模型  底层B：金融语言大模型
   （时序地基）          （语言地基）
```

#### 底层 — 两个大模型地基

| 地基 | 模型 | 作用 | 学会什么 |
|------|------|------|---------|
| **A（时序）** | Kronos/PatchTST/iTransformer | K线预测：量价关系自动挖掘 | 自己学会量价关系，甚至悟出缠论级别的结构 |
| **B（语言）** | FinGPT/PIXIU/DISC-FinLLM | 金融文本理解：新闻/研报/宏观/基本面 | 学会金融语言的深层逻辑 |

底层模型是**开源、可本地部署、可微调**的预训练大模型。不需要用户从零训练，只需要用自己的数据微调适配。

#### 中层 — 模因层（用户的方法论体系）

**这是用户自己写的全部代码所在层。不是因子库，是方法论体系。**

传统量化把中层理解为"XGB等多因子模型"，但本架构中的模因层是指用户的**方法论框架**：

- 缠论结构（位置+状态的判断）
- ★买闭环、量领选股
- Dorian六步叙事法、变量传导推演
- ABCD级别、两轴决策、7维信号质量
- EXPMA/MACD/CCI 等经验性信号

**这层的作用**：在底层两个大模型的基础上，对预测结果做**方向确认和信号增强**。

| 场景 | 底层大模型 | 中层方法论 |
|------|-----------|-----------|
| 信号会错但不会太离谱 | ✅ 用统计兜住概率下限 | ✅ 人工经验做二次过滤 |
| 对的时候拿不住 | ✅ 统计信心撑住持有 | ✅ 方法论确认趋势延续 |

底层大模型学"规律"，中层方法论做"判断"和"确认"。二者不是竞争关系。

#### 上层 — 图谱层

以**资金流动性/资本流向**为核心枢纽的金融关系图谱。

底层B（语言LLM）理解了金融信息之后，用中层的方法论框架（Dorian六步、变量传导）去建立实体之间的关系网络，最终形成一个能回答"钱流到哪里了"的知识图谱。

**为什么资金流动是核心枢纽**：
> 所有的因果导向最后的反应就是资金的流动，从哪些方向流向另外一些方向。而这个流动的背后是每一个人每一个机构针对各种各样的消息面做出的决策。

资金流串联了：消息面、基本面、研报、纪要、短线情绪。它不是跟其他因子并列的维度，而是**所有因子的共同出口**。

### 9.4 关键认知矫正

1. **模因不是因子库** — 模因是用户的交易方法论（可复制的框架），不是XGB多因子。把方法论当"因子"是对本架构最大的误解。
2. **底层不需要从零训练** — Kronos/FinGPT 都是开源预训练模型，用户只需要微调适配自己的数据和场景。
3. **不是取代现有系统** — 底层模型负责降低"离谱预测"的概率 + 提高持有信心，中层用户方法论负责确认和过滤，上层图谱负责方向筛选。三层各有不可替代的价值。
4. **Claude不能替代底层模型** — Claude是通用对话模型（API），不能本地部署、不能微调、不能做K线时序预测。Claude在这个架构中的角色是**中上层的分析引擎**（读取图谱、分析叙事、生成判断），不是底层。

### 9.6 Kronos-mini 全面验证结果（2026-06-18）

**背景**: 验证Kronos-mini (4.1M参数) 能否作为时序模因图谱的"底层"模型。

**测试方法**: 8个标的 × 3组参数(T=0.8/1.2/1.3) × 10次均匀采样 = 240次预测。叠加缠论笔方向一致性检测、参数偏误检测、结构约束违反率检测。

**核心结论**: Kronos-mini 不适用于当前系统。关键证据:

| 指标 | 数值 | 含义 |
|------|------|------|
| 方向正确率 | 50±2% | 猜硬币水平 |
| 看空偏误 | -10%~-37% | 系统性低估涨 |
| 笔方向感知 | 16%区分度 | 有但不实用 |
| 最优参数 | T=1.3, p=0.95, s=3 | 偏误最小化 |

**根因**: 4.1M参数不足以从裸K线中提取缠论级别的结构信息。模型学到了"噪音的形状"而非可交易的结构。

**方向修正**: 时序模因图谱的底层不应依赖纯时间序列小模型。替代方案见用户对话——用已有缠论信号标注训练数据，对Kronos做supervised fine-tuning，加入结构识别头（auxiliary loss）。
