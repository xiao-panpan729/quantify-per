## 一、一句话

> **板块K线图上标注每一波行情区间：时间窗口、龙头股（龙1/龙2）、涨幅、参与广度**

## 三、实现逻辑

```
输入: 一个板块概念的所有成分股日线（通达信880XXX概念板块）

Step 1: 扫描异常涨幅集群
  滚动窗口（20日/5日）:
    - 板块指数涨幅超过阈值? (1月>20%, 1周>15%)
    - 板块内多少只成分股同期异常? (参与广度)
  如果板块+成分股同时异常 → 标记为"潜在节点区间"

Step 2: 精准定位节点边界
  在潜在区间内，找板块指数的最优买入点:
    - EXPMA金叉? 放量突破? 结构双底?
    - 用已有信号CSV的数据即可判断
  节点范围 = 从启动日到趋势结束（expma死叉/放量滞涨）

Step 3: 识别龙头股
  在节点范围内:
    - 找涨幅最大且持续时间最长的成分股 = 龙1
    - 找第二梯队 = 龙2
    - 计算每只领涨股的持续天数
  （同花顺的"龙1XX(4天)"就是这么来的）

Step 4: 节点上下文标注（逆推）
  节点时间窗口内:
    - 流动性压力（liquidity_monitor 回溯）
    - 板块势能排名（sector_momentum 回溯）
    - US映射（us_macro_sensitivity 回溯）
    - 事件标签（AI问：这个板块这个时间在涨什么？）

Step 5: ★买映射
  你的★买入场在不在节点窗口内?
  在 → 信号有板块效应支撑，置信度上调
  不在 → 信号是独立行为，需要额外确认

Step 6: 贝叶斯收缩
  按节点的"流动性+板块效应强度+龙头级别"分组
  算各组★买胜率 → 置信度修正系数

## 四、优先级

第一优先级（概念板块节点地图）:
  通达信880XXX概念板块（你sector_momentum已经在用的）
  黄金概念、半导体、光伏、新能源车等269个板块
  逐个生成节点地图，保存为 node_map_{concept_code}.json

第二优先级（你的宇宙内的标的映射）:
  你跟踪的14只标的 + 量领宇宙的标的
  看它们分别属于哪些概念板块的节点
  自动标注：当前是否处于某个板块节点窗口内

第三优先级（贝叶斯收缩）:
  基于节点标签分组 → 修正系数
