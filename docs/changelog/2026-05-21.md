## 信号密度改为波段计数（indicators.py signal_quality）
- **原因**：全窗口平均（"3次/448根K线"）没有意义，密度应该按"这一段下跌/上涨"算
- **买侧**：`buy_wave_start` = 最近一个★卖或死叉的位置，只计此之后的★买
- **卖侧**：`sell_wave_start` = 最近一个★买或金叉的位置，只计此之后的★卖
- **阈值**：≥3次/波=密集+1.5 / 2次/波=连续+1.0 / 1次/波=单次+0.5 / 0=0
- **标签**：从"X次/NNN根K线"改为"X次/波"

## MA排列评分重构 + CYCLE净值制 + 量能对称化

### MA排列：链式断链 → 浮点扣分制（indicators.py judge_trend）
- **旧逻辑**：5对MA关系逐对检查，任何一对断裂即归零，极端粗糙（5死叉10直接0分）
- **新逻辑**（总分6分不变，浮点输出）：
  - 5/10区(1分): close>5MA(0.5) + 5MA>10MA(0.5)
  - 10/20区(1分): close>10MA(0.5) + 10MA>20MA(0.5)
  - 20/60区(1分): close>20MA(0.5) + 20MA>60MA(0.5)
  - 60/120区(1分): 60MA>120MA
  - 120/250区(1分): 120MA>250MA
  - 满分加成(+1): 8个子项全满足
- **效果**：短线价格破位早于均线死叉扣分，长线排列完好时不会因短线弱全部归零

### CYCLE：buy_level单侧 → 8维净值制（indicators.py signal_quality）
- **旧逻辑**：cycle_score只取daily_buy_level，忽略空头信号
- **新逻辑**：net_score = buy_level - sell_level，每维度买侧正分 + 卖侧负分
  - 维1: ★买密集(+0.5~1.5) vs ★卖密集(-0.5~-1.5)
  - 维2: 金叉跟随(+0.3~1.5) vs 死叉跟随(-0.3~-1.5)
  - 维3: 缠论底部抬升(+1.0) vs 顶部下移(-1.0)
  - 维4-6: 自带分类，不改
  - 维7: 量能 — 买侧地量(+1.5) vs 卖侧放量(-1.5) ★新增
  - 维8: 趋势线突破，不变
- 映射: net>=6→4分, >=4→3分, >=2→2分, >=0→1分, <0→0分

### 卖侧量能三维度（indicators.py signal_quality + signal_engine.py）
- 对称买侧地量堆(+1.5)：
  - 7a': 百日高量+★卖附近 → -0.5（新增 vol_hhv100 字段）
  - 7b': 放量堆存在于波内 → -0.5（新增 vol_放堆 字段）
