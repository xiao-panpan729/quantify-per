## 一、核心理念

quantify-per（量化引擎）和 knowledge-hub（叙事引擎）是**平级系统**，不是上下级：

```
quantify-per                    knowledge-hub
  │                                │
  │ 价格 → 信号 → 结构 → 判断        │ 事件 → 实体 → 关系 → 演化
  │                                │
  │ "市场正在干什么"                 │ "为什么这么干"
  │                                │
  └────────────┬───────────────────┘
               │
         战役级交汇点
         （两个系统的输出在这里碰撞 → 综合决策）
```

**核心洞察**（来自 2026-05-15 实战）：
- 上周五大盘下跌时，sz159740 高开 → 如果不看宏观环境，只看技术信号，可能错误加仓
- 如果将"大盘判定放在第一位"，高开后连续5分钟阴线时不会加仓甚至减仓
- **当技术面和叙事面对不上时，相信叙事** → 小级信号不足以对抗大叙事

**MiroFish 启发**：
- MiroFish 将"种子材料→知识图谱→Agent 群体→涌现预测"做成管道
- 我们不需要模拟 Twitter，但需要**在时间轴上组织事件→关系→影响力**
- 核心偷学：GraphRAG 做实体提取+关系构建，按时间线组织

## 二、横轴+纵轴 结构设计

### 横轴 = 时间线
一条连续的、不可逆的时间流。每个时间节点挂着当时的真实事件记录（**事后不修改**）：

```
2025-Q1  →  2025-Q2  →  2025-Q3  →  2025-Q4  →  2026-Q1  →  2026-05-18
  │           │           │           │           │           │
 美联储      降息周期    大选       关税升级    A股反弹    当前位置
 表态偏鹰    开启        不确定性    加码       创业板突破
```

### 纵轴 = 多事件联动
同一时间节点上，多个事件同时存在。四种关系：

| 关系类型 | 解释 | 例子 |
|----------|------|------|
| **放大** | 两个同向事件叠加，影响力 > 各自单独 | 美股跌 + 美联储鹰派 = 双重利空 |
| **对冲** | 两个反向事件抵消，方向待定 | 美股跌 + A股政策利好 = 对冲 |
| **主导** | S级事件压倒低级别事件 | 美联储降息压倒当日小作文 |
| **诱变** | 小事件触发大事件连锁反应 | 某基金爆仓 → 板块恐慌 → 流动性危机 |

## 三、三层叙事分类

### 第一层：宏观叙事（S/A 级事件）

| 等级 | 影响力 | 时效 | 例子 |
|------|--------|------|------|
| **S** | 改变市场方向 | 周-月 | 美联储加息/降息、重大政策转向、地缘冲突 |
| **A** | 短期冲击+方向确认 | 天-周 | CPI/PMI/非农数据、中央经济工作会议 |
| **B** | 板块级别影响 | 天 | 行业政策、监管部门发声、大行研报 |
| **C** | 日内波动 | 小时-天 | 消息面发酵、小作文、市场传言 |

**事件记录格式**：
```
时间 | 事件摘要 | 等级 | 预期方向 | 受影响板块 | 置信度 | 事后验证
```

### 第二层：板块叙事

热度-强度矩阵：

| | 强度强（有实质进展）| 强度弱（纯炒作）|
|---|---|---|
| **热度高** | ⭐ 主线板块 | 烟花板块 |
| **热度低** | 潜伏板块 | 冷板块 |

叙事阶段（跟现有 ABCD 级别对应）：
- **萌芽期** → 启动期 → 主升期（A/B级做多窗口）
- **分歧期** → 衰退期（C/D级回避）

### 第三层：个股叙事

每只标的一条事件时间线：
- 基本面突变（业绩、管理层、大订单）
- 资金面变化（机构增减持、北向资金）
- 结构点（突破、破位、地量、天量——直接读 quantify-per 输出）
- 事件驱动（纳入指数、解禁、分红）

### 9.1 第一篇：什么是"框架"？（2026-02-21）

**核心论点**：AI 不会自动让你变强——它只会放大你已有的认知结构。

#### 框架定义
> 宏观框架不是简单市场预测。它是一个变量关系图。一张向量表，拓扑。

#### 黄金四变量传导模型
```
实际利率 → 持有黄金成本上升 → 黄金承压
美元走强 → 黄金计价压力 → 黄金回调
流动性扩张 → 风险偏好变化 → 贵金属受益
地缘风险 → 避险需求 → 黄金受益
```

四个变量之间有传导路径，不是独立作用。

#### AI 三层提问升级
```
L1 信息消费:  "黄金怎么看？" "美股会不会跌？"
             → 随机化碎片化的AI回答，没有底层逻辑

L2 变量拆解:  "拆解黄金的核心驱动变量，分长期中期短期"
             → 得到变量列表，开始建立结构

L3 结构推演:  "如果收益率上行但美元走弱，市场可能出现什么结构？"
             → AI 帮你推演情境，建立 Scenario Thinking
```

#### 实战对比：美债收益率上行案例

| | 普通问法 | 高阶问法 |
|---|---|---|
| 问题 | "美债涨了，股市会跌吗？" | 本次收益率上行是通胀驱动还是财政供给驱动？ |
| | | 实际利率是否同步上行？ |
| | | 美元是否跟随走强？ |
| | | 风险资产是同步下跌还是分化？ |
| 结果 | 猜方向 | 推演情境 |

#### 三个直接可用的 Prompt 模板
1. **变量拆解**: `请拆解【资产名称】的核心驱动变量，并分为长期、中期、短期三类。`
2. **情景推演**: `如果【变量A】上升，同时【变量B】下降，市场可能出现哪些路径？`
3. **失效验证**: `当前判断的最大风险点在哪里？哪些数据会推翻这个逻辑？`

#### '失效机制'概念
> 交易最重要的一步是：什么时候我错了？
> - "如果黄金逻辑失效，可能是哪些变量发生了变化？"
> - "在什么情况下，当前多头逻辑会被推翻？"
> 你不是在强化信念。你是在构建风险边界。

#### 月度练习计划
- 第一周：对三个资产做变量拆解
- 第二周：对两个宏观事件做情景推演
- 第三周：对自己的持仓做一次完整的失效测试
- 一个月后：判断方式会明显不同

#### 核心金句
> - "AI不会让普通交易者突然赚钱。但它可以让你：更少情绪化决策、更清晰逻辑结构、更明确风险边界。"
> - "真正的差距，从来不是信息。而是：你是否拥有自己的框架。"
> - "如果没有框架，AI只是噪音。如果有框架，AI将成为你的强化思维外骨骼。"

### 9.2 第二篇：怎么建立"流程"？（2026-04-02）

**核心论点**：专业机构真正的优势从来不是信息，而是有一条固定的 Research Pipeline。

#### 错误流程 vs 专业流程（框架对比）

```
错误流程（情绪驱动）:
  新闻 → K线 → 情绪判断 → 下单 → 后悔

专业流程（结构驱动）:
  Macro Event → Variable Impact → Market Structure
    → Scenario Tree → Trade Expression → Risk Boundary
```

本质区别：**是否结构化，流程化？**

#### 六步投研管道 ★（知识库核心模板）

```
[Step 1] 事件拆解
  不问："市场会不会跌？"
  问：  "请拆解【这个宏观事件】影响哪些关键变量。"
  输出: 实际利率、美元、流动性、风险偏好、信用利差

[Step 2] 变量传导路径
  问：  "这些变量之间如何传导？"
  输出: 实际利率↑ → 估值压力 → Growth下跌
        美元走强 → 全球流动性收紧 → EM承压
  本质: 市场不是涨跌，是变量互动

[Step 3] 历史参考
  问：  "历史上类似环境市场如何表现？"
  输出: 2013 Taper Tantrum / 2018 加息周期 / 2022 QT
  作用: 建立历史坐标系，减少情绪判断

[Step 4] 情景推演
  问：  "如果收益率上行但美元不涨可能意味着什么？"
  推演: 财政供给驱动 / 资本流变化 / 期限结构压力
  关键: 建立 Scenario Thinking，而不是方向赌注

[Step 5] 交易表达
  问：  "如果此逻辑成立有哪些交易表达？"
  输出: Short duration / Long USD / Short growth / Long vol
  关键: 连接"研究 → 交易"

[Step 6] 失效条件
  问：  "什么情况说明这个逻辑错了？"
  定义: 收益率回落 / 美元走弱 / 信用利差收窄
  本质: 专业交易者不是更会赚钱，是更会定义错误
```

#### 每日执行模板

每天只做三件事：
1. 选一个宏观事件
2. 走一遍六步流程（问/追问/记录）
3. 记录结论

#### 研究复利时间线
- 一周后：看市场开始有结构
- 一个月后：不再乱看新闻，有自己的主见
- 三个月后：形成自己的认知系统
- 这叫：**Research Compounding（研究/认知复利系统）**

#### 通用 Prompt 模板
`请按以下结构分析【宏观事件】：1.关键变量 2.变量传导路径 3.历史类似案例 4.可能市场结构 5.交易表达方式 6.失效条件`

#### 核心金句
> - "市场从不奖励知道最多信息的人。市场奖励结构最清晰的人。"
> - "没有流程：AI是噪音放大器。有流程：AI是认知外骨骼。"
> - "普通交易者和机构之间真正的差距不是资金，更不是信息。而是：是否拥有自己的研究流程。"

### 9.3 第三篇：同一个思维用到财报分析（2026-04-09）

**核心论点**：99% 用 AI"看"财报（信息消费），1% 用 AI"拆"财报（分析能力）。差距不在 AI，在有没有分析系统。

#### 99% vs 1% 框架对比

| | 99%（信息消费） | 1%（分析能力） |
|---|---|---|
| 做什么 | 总结、翻译、提取、解释 | 拆变量、拆结构、找预期差 |
| 问题 | "帮我总结财报" | "拆 revenue driver → segment growth → margin变化 → guidance变化 → 市场预期差" |
| 结果 | 看完了，判断水平没提高 | 建立了判断结构 |
| AI 角色 | 替你阅读 | 参与你的分析流程 |

核心洞察：**AI 不会自动给你分析能力。它只会放大你已有的认知结构。如果你没有结构，它只会帮你更快浏览。**

#### 四层财报能力框架（完整版）

```
Financial-Analysis-Core/
├── Layer_1_Read_the_Numbers        ← 读数据层
│   ├── Revenue                     ← 收入
│   ├── Margin                      ← 利润率
│   ├── EPS                         ← 每股收益
│   ├── Cash flow                   ← 现金流
│   └── Balance sheet               ← 资产负债表
│
├── Layer_2_Explain_the_Drivers     ← 解释驱动层
│   ├── Why growth changed          ← 增长为什么变？
│   ├── Why margin changed          ← 利润率为什么变？
│   ├── Why cash changed            ← 现金为什么变？
│   ├── What management changed     ← 管理层改了哪些策略？

### 9.4 第四篇：从流程升级到系统 ★★★ (2026-04-12)

**核心论点**：大多数人死在"以为自己有流程"。真正的差距：**分析能不能持续运行**。

#### 流程 ≠ 系统

```
流程（手动模式）:
  宏观事件 → 变量拆解 → 传导路径 → 历史参考 → 情景推演 → 交易表达 → 失效条件
  问题：每次从零开始，依赖记忆和意志力；今天分析明天忘

系统（自动模式）= 流程 + 三层核心:

  ┌── 记忆层（Memory）
  │   没有记忆 → 每天从零开始
  │   有记忆 → 情景库、历史记录 → 认知数据库
  │   AI 作用：检索历史结构、做对比、提炼规律
  │
  ├── 触发层（Trigger）
  │   没有触发 → 被市场拖着走，随机看新闻
  │   有触发 → 定义"值得分析" → 利率变动超阈值/汇率突破区间/商品异常
  │   AI 作用：监控变量、自动标记异常
  │
  └── 执行层（Execution）
      没有执行 → 手动跑流程，依赖意志力
      有执行 → 固定管道、自动化 → 不依赖情绪
      AI 作用：快速建模变量、生成因果链、推演情景
```

#### 七层系统架构（完整版）

```
[数据输入层]
  固定输入：利率(美债) / 美元(DXY) / 商品(油/金) / 信用利差 / 宏观事件(央行/政策)
  AI 作用：压缩信息、提取变量、标准化输入
  你不是在看新闻。你是在：喂系统。
        ↓
[事件触发层]
  定义"什么叫值得分析"：
    利率变动超过某阈值 / 汇率突破区间 / 商品异常波动 / 流动性变化
  触发 → 才分析。没有触发机制 → 被市场拖着走。
  触发层是领先市场的关键。
        ↓
[宏观研究流程（六步）]
  事件拆解 → 变量传导 → 历史参考 → 情景推演 → 交易表达 → 失效条件
  当流程被触发后才运行，它才真正有价值。
  你不再猜涨跌。你在运行：结构模型。
        ↓
[情景库]
  记录：事件 / 变量 / 情景 / 结果
  变成：你的"认知数据库"
  AI 作用：检索历史结构、做对比、提炼规律
  这一步决定你是否会进化。
        ↓
[交易映射层]
  强制：情景 → 交易表达
  否则你只是一个会分析但赚不到钱的人。
        ↓
[风险控制层]
  不问："会不会涨？"
  问：  "我什么时候错？"
  定义：失效条件 / 时间窗口 / 结构变化
  AI 作用：找漏洞、推演反向路径
  风险不是保护，是让你活下来。
        ↓
[反馈系统]
  每次之后必须问：哪里错了？哪个变量关键？哪个逻辑有效？
  然后写回系统。
  你优化的不是交易。是：你的判断函数。
```

## 十、AI 提问方法论与问题分拆模板

### 10.1 核心原则

**AI 不会自动给你分析能力——它只会放大你已有的认知结构。**

- 如果你没有框架，AI 只是更快的噪音放大器
- 如果你有框架，AI 会成为认知外骨骼
- 差距不在 AI，在"你知不知道该问什么"

### 10.2 三层提问升级路径

| 层级 | 问法 | 例子 |
|------|------|------|
| **L1 信息消费** | "XXX怎么样？会不会涨/跌？" | "黄金怎么看？" ← AI 给随机化碎片化回答 |
| **L2 变量拆解** | "拆解XXX的核心驱动变量，分长期/中期/短期" | "拆解黄金的核心驱动，区分长期变量和短期变量" |
| **L3 结构推演** | "如果变量A上升但变量B不跟，意味着什么？什么情况下这个逻辑失效？" | "如果收益率上行但美元走弱，市场可能出现什么结构？" |

### 10.3 10 条每日可用的 AI 投研 Prompt

这些是可以直接用于知识库建设的提问模板：

1. **宏观事件拆解** — `请拆解【某个宏观事件】影响的关键变量，并按重要性排序`
2. **变量传导路径** — `这些变量之间如何传导？请用因果链形式表达`
3. **市场结构影响** — `这些变量变化，会如何影响：利率、汇率、股市、商品`
4. **历史类似情景** — `历史上是否出现过类似结构？对应市场表现如何？`
5. **情景推演（关键）** — `如果【变量A发生】但【变量B不跟随】，意味着什么？`
6. **市场在交易什么** — `当前市场最可能在交易的核心逻辑是什么？是否与表面新闻一致？`
7. **交易表达** — `如果上述逻辑成立，对应的交易路径是什么？（标的+方向+逻辑）`
8. **反向推演** — `如果这个逻辑是错的，最可能错在哪里？`
9. **风险失效条件** — `哪些变量变化，说明这个交易逻辑已经失效？`
10. **压缩总结（做笔记用）** — `请用3句话总结：当前结构、核心驱动、交易方向`

### 10.4 问题分拆的结构化方法

面对任何一个宏观事件/市场变化，不直接问"怎么看"，而是**按固定结构拆解**：

```
[事件发生]
    │
    ├── Q1: 影响哪些变量？（列出全部，按重要性排序）
    │
    ├── Q2: 变量之间怎么传导？（画因果链）
    │        实际利率 → 估值压力 → Growth↓
    │        美元走强 → 全球流动性收紧 → EM承压
    │
    ├── Q3: 历史上类似情况怎么走的？（找参照系）
    │        2013 Taper Tantrum / 2018 加息周期 / 2022 QT
    │
    ├── Q4: 分歧情景是什么？（不赌单边）
    │        情景A：变量B不跟 → 可能意味着...
    │        情景B：变量C超预期 → 可能导致...
    │
    ├── Q5: 怎么表达成交易？（研究→交易）
    │        做多/做空什么？用什么工具？时间窗口？
    │
    └── Q6: 什么条件下我错了？（风险边界）
             哪些变量变化？多长时间没兑现？结构怎么变？
```

### 10.5 认知复利模型（Research Compounding）

```
每天3件事:
  1. 选一个宏观事件
  2. 走一遍六步流程（拆变量→传导→历史→情景→交易→失效）
  3. 记录结论到知识库

1周后: 看市场开始有结构，不再乱刷新闻
1月后: 有自己的主见和判断路径
3月后: 形成自己的认知系统
1年后: 拥有一套可复用的研究引擎
```

**这与我们做 PE 轨迹化的动机相同**——不满足于"今日PE=0.78"的快照，而要看 PE 在 20 天里的演化曲线。认知复利的前提是有**连续的时间序列记录**。

### 关键设计决策

**1. 知识库不做"通用百科"，做"交易决策支持"**

不是收集所有资讯，而是只记录"影响交易判断的变量和关系"。每条笔记末尾必须有：
```markdown
**交易含义**: 这个信息对当前的持仓/关注标的意味着什么？
**失效条件**: 什么情况下这个判断不再成立？
```

**2. 触发层是知识库和量化系统的桥梁**

quantify-per 的 signal_engine 负责技术触发（CCI极值、★买卖），knowledge-hub 负责叙事触发（宏观事件、板块轮动）。两周碰一次，对不上时以叙事为准。

**3. 反馈系统是认知复利的发动机**

每次交易判断后必须写回 decision log，一个月后回头看：判断命中率、失效条件触发率、最常出错的变量。这不是回测统计（回测只看价格），而是**判断力审计**。
