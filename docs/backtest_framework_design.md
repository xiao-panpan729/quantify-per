# 信号回测框架设计文档

## 1. 目标

验证 CCI + 分时出击 + EXPMA 信号体系的有效性，找出：
- 哪些信号模式胜率高？
- 哪些级别（5/15/30/60分钟）的信号更可靠？
- 不同趋势环境（日线多头/空头）下信号表现如何？
- 买入后持有N天的收益率分布

## 2. 核心概念

### 2.1 信号定义

| 信号类型 | 定义 | 级别 |
|---------|------|------|
| ★买 | CCI触及-200/-250/-300极限 + 分时出击买入信号 | 5/15/30/60分钟 |
| ★卖 | CCI触及+200/+250极限 + 分时出击卖出信号 | 5/15/30/60分钟 |
| 金叉 | EXPMA12上穿EXPMA50 | 各周期 |
| 死叉 | EXPMA12下穿EXPMA50 | 各周期 |
| 背驰 | CCI与价格背离 | 各周期 |

### 2.2 闭环信号

**买入闭环**: CCI负极限 + ★买 + 金叉（同一周期内出现）
**卖出闭环**: CCI正极限 + ★卖 + 死叉（同一周期内出现）

### 2.3 趋势环境

| 等级 | 条件 | 对信号的影响 |
|------|------|-------------|
| A最强 | 股价在EXPMA白线上方 + MACD>0 | 小级别回调即买点 |
| B次强 | 股价在白线-黄线间 + MACD>0 | 小级别信号有效 |
| C偏弱 | 黄线下方但MACD>0 | 需大级别信号确认 |
| D弱势 | MACD<0 | 小级别反弹不可信，等大级别底部结构 |

## 3. 回测数据需求

### 3.1 输入数据

```
signals/tracking/{code}/
  ├── daily_signals.csv      (日线: MACD, EXPMA, CCI)
  ├── min5_signals.csv       (5分钟: 分时出击, EXPMA, CCI)
  ├── min15_signals.csv      (15分钟)
  ├── min30_signals.csv      (30分钟)
  └── min60_signals.csv      (60分钟)
```

### 3.2 需要记录的字段

对于每个信号出现点：
- 标的代码、名称
- 信号日期时间
- 信号周期（5/15/30/60分钟）
- 信号类型（★买/★卖/金叉/死叉/背驰）
- 当日趋势等级（A/B/C/D）
- 当日日线MACD状态（多头/空头/0轴）
- 信号出现时收盘价
- 信号出现后1/3/5/10/20日收盘价
- 信号出现后最大涨幅、最大跌幅
- 是否形成闭环（同一周期内是否同时出现CCI极限+出击+交叉）

## 4. 回测统计维度

### 4.1 按信号类型统计

| 统计项 | 说明 |
|--------|------|
| 信号次数 | 该信号总出现次数 |
| 胜率 | 信号后N日上涨的比例 |
| 平均收益 | 信号后N日平均收益率 |
| 最大收益 | 信号后N日最高收益率 |
| 最大亏损 | 信号后N日最大亏损率 |
| 盈亏比 | 平均盈利 / 平均亏损 |

### 4.2 按级别统计

分别统计 5分钟 / 15分钟 / 30分钟 / 60分钟 的信号表现。

### 4.3 按趋势环境统计

分别统计 趋势A/B/C/D 环境下的信号表现。

### 4.4 闭环 vs 非闭环

对比完整闭环信号和部分信号的表现差异。

## 5. 关键假设

1. **买入时点**: 信号出现当日收盘价买入（或次日开盘价）
2. **卖出时点**: 持有N天后卖出，或出现反向闭环信号时卖出
3. **手续费**: 暂不考虑（或固定扣0.1%）
4. **滑点**: 暂不考虑

## 6. 输出报告

### 6.1 信号有效性排行榜

```
Top 10 最有效信号（按胜率+盈亏比综合排序）:
1. 60分钟★买+金叉 + 趋势A/B → 胜率72%, 平均收益5.3%
2. 15分钟★买+金叉 + 趋势B → 胜率68%, 平均收益3.8%
...
```

### 6.2 信号失效模式

```
最容易失败的信号:
1. 5分钟★买 + 趋势D → 胜率32%, 平均亏损-2.1%
   → 结论: 大趋势空头时，5分钟反弹不可信
2. 30分钟★买 但无金叉确认 → 胜率41%, 平均收益0.3%
   → 结论: 缺少金叉的★买信号可靠性大幅下降
```

### 6.3 最佳持有周期

```
不同信号的 optimal hold period:
- 5分钟闭环信号: 最佳持有1-3天
- 15分钟闭环信号: 最佳持有3-5天
- 60分钟闭环信号: 最佳持有5-10天
```

## 7. 实现路径

### Phase 1: 数据收集（1-2周）
- 每日运行后自动记录信号出现记录到 `backtest/signals_history.csv`
- 字段: date, code, period, signal_type, trend_level, close, ...

### Phase 2: 回测计算（1周）
- 读取历史信号记录，计算每个信号后N日收益
- 生成统计报告

### Phase 3: 策略优化（持续）
- 根据回测结果调整信号权重
- 优化买入/卖出规则

## 8. 与每日报告的整合

每日报告中增加：
- "历史同类信号表现": 当出现某信号时，自动提示"历史上该信号在趋势X环境下，5日后平均收益Y%"
- "信号置信度评分": 基于回测数据给当前信号打分（高/中/低）
