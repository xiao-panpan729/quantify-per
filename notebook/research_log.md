## 实验 #0: 框架方向确认 (2026-06-02)

### 背景

笔记本系统骨架已搭建完成：技能系统 → 预测卡 → 验证引擎 → 案例库 → 反馈回路（[notebook_design.md](notebook_design.md)）。现有技能均为规则驱动（confidence 固定1.0）。下一个方向是从笔记本项目出发做深度学习研究与开发。

### 讨论结论

**核心框架**: 缠论数学结构（骨架）+ 多维度评分（血肉）→ DL学习结构→结果的映射规律

类比：AlphaGo 策略网络 — 人类棋谱（先验知识）预训练 + 自我对弈（数据驱动）强化

**数据规模**: 73只标的有信号数据（非仅14只跟踪标的），每只几百~2600条日线，总计约15万+样本

**技术路线**: 先验知识约束 → DL数据驱动修正。不推翻规则，用DL给规则打概率。

### 特征设计（v0草案）

**缠论骨架（6维）**:
| 维度 | 编码 | 来源 |
|------|------|------|
| 趋势方向 | 1(上涨笔延伸)/0(盘整)/-1(下跌笔延伸) | positions.py |
| 笔方向 | 1(向上笔)/-1(向下笔)/0(分型未确认) | czsc |
| 笔强度 | 当前笔涨跌幅% | cxt_bi_zdf |
| 日线中枢位置 | 1(上方)/0(内部)/-1(下方) | positions.py |
| 30分中枢位置 | 1(上方)/0(内部)/-1(下方) | positions.py |
| 背驰状态 | 1(底背驰)/0(无)/-1(顶背驰) | signals.py |

**日×30双级联立**: 3×3 = 9 种组合状态

**评分血肉（8维）**: 趋势评分(0-14) / ABCD级别 / CCI / EXPMA位置 / ★买密度 / ★卖密度 / 量能比 / PE水平

**交互特征**: 趋势评分×日线中枢位置 / 趋势评分×笔方向 / ABCD×日线中枢位置

### 标签编号体系（R0-R5）

| 编号 | 含义 | 初始使用 |
|------|------|---------|
| R1 | 信号群后N天涨超5%且回踩不破 | 正样本 |
| R2 | 涨了但幅度不够 | 暂归R0 |
| R3 | 横盘没变化 | 暂归R0 |
| R4 | 跌了但结构没破坏 | 暂归R0 |
| R5 | 跌了且跌破关键位置 | 负样本 |
| R0 | 不确定/中间状态 | 不参与训练 |

初始做二分类（R1 vs R5），后续迭代细化。

### "信号群"概念

关键发现：多个★买/金叉/背驰信号活在同一个缠论结构里（如同一笔下跌），它们不是独立信号而是一个"信号群"在反复确认同一个底部。

- 群内信号次数 = 确认强度
- 群的质量 ≠ 单个信号质量
- 标签应贴在"信号群+缠论结构"的组合上，非单个信号

| 阶段 | 模型 | 目标 |
|------|------|------|
| 第一步 | XGBoost | 快速验证特征有用性（秒级训练，可解释特征重要性） |

先XGBoost而非直接DL的原因：特征21维左右，结构化表格数据，XGBoost通常不输DL且能输出特征重要性排名。

### 背景

实验#0 确认了"缠论骨架+评分血肉→DL数据驱动"的框架方向。第一步按计划用 XGBoost 快速验证特征有用性。

实现了独立探索脚本 [notebook/explore_xgb.py](notebook/explore_xgb.py)：
- 77只标的（量领宇宙全量，非仅14只跟踪标的）
- 日线+30分钟双级联立
- 缠论6维：笔方向/笔力度/趋势方向/日线中枢位置/30分中枢位置/背驰状态
- 评分8维：趋势评分/ABCD级别/CCI/EXPMA位置/★买密度/★卖密度/VR60/PE水平
- 交互3维：趋势评分×中枢位置 / 趋势评分×笔方向 / ABCD×中枢位置

### 开发过程中的 bug 修复

| 日期解析 | 全量 RawBar dt 相同 | adapter.py 对 float date "20150902.0" 解析失败 → NaT → datetime.now() | 自写 `rows_to_bars()` 绕过 adapter，手动清理日期格式 |

**核心发现**：

1. **笔方向是压倒性特征**（59.4%）。当前标签"未来5天涨5%"本质上在捕捉趋势延续——向上笔中的bar更容易在5天后涨超5%。这不是泄露，是缠论结构本身的预测力，但可能有微弱前视偏差（bi确认滞后2-3根K线）。

2. **笔力度第二**（20.0%）。涨跌幅大的笔，方向持续性更强——符合直觉。

4. **背驰特征完全失效**（0.0%）。`get_bs_points()` 被 try/except 吞掉后大部分标的返回空字典。需要单独的背驰计算路径，不依赖 SMA 版本的三买三卖函数。

5. **评分特征中 EXPMA位置 最有用**（5.8%）。白线/黄线/下方的三分法比ABCD级别（0.9%）更直接有效。

## 实验 #2: 20天标签 + 中枢细化 + 背驰修复 (2026-06-02)

### 背景

实验#1 暴露了三个问题：
1. 5天标签太短，笔方向(T+0已知)占59%重要性，标签高度依赖趋势方向本身
2. 中枢只有粗糙三分类，没体现"第几个中枢"+"连续位置"
3. 背驰因 SMA 崩溃 + 方向值不匹配完全失效

本轮按用户要求：标签改为20天、中枢加序号+连续位置、背驰用笔数据自己算。

### 改动内容

| 改动 | 旧 | 新 |
|:--|:--|:--|
| 标签周期 | 5天 | **20天** (未来20天涨>5%=R1, 跌>5%=R5) |
| 中枢特征 | zs_daily(离散三分类) | +zs_index(第几个中枢) +zs_position(连续-1~1+) |
| 背驰 | 空(0%) | **连续动量背驰分数** (-1~1, 每笔比较前一同向笔) |
| 交互特征 | 3个 | +ix_zs_position_x_score +ix_zs2_x_div(第二中枢×背驰) |
| 方向值匹配 | "Up"/"Down" | **"向上"/"向下"** (czsc Direction.value 实际值) |

### 关键 bug 发现

**czsc Direction 枚举值**：`Direction.Up.value = '向上'`，`Direction.Down.value = '向下'`。之前的 `("Up", "Down", "上", "下")` 只有子串"上"/"下"能匹配"向上"/"向下"，精确匹配 `in ("Down", "下")` 完全失效。这就是实验#1背驰全零的根因。

### 结论

**77只全量运行**（2026-06-02）：

| 指标 | 实验#1 (5天) | 实验#2 (20天) | 变化 |
|:--|:--|:--|:--|
| 有效样本 | 64,262 | 109,960 | +71% (更少R0) |
| AUC | 0.9480 | 0.8582 | -9% (20天更难) |
| 准确率 | 89.0% | 77.1% | -12% |
| 缠论占比 | 82.9% | 71.3% | -11.6% (趋势特征主导减弱) |
| 评分占比 | 13.4% | 19.4% | +6.0% (长周期评分更有用) |
| 交互占比 | 3.7% | 9.3% | +5.6% (新交互特征有效) |

**特征重要性排名**：

| 排 | 特征 | 重要性 | 变化 | 含义 |
|:--|:--|:--|:--|:--|
| 1 | cl_trend_dir | 43.3% | -16.1% | 笔方向仍是王者但下降明显 |
| 2 | cl_bi_strength | 12.3% | -7.7% | 笔力度 |
| **3** | **cl_divergence** | **7.2%** | **+7.2%** | **背驰复活, 排第三!** |
| 4 | sc_expma_pos | 5.8% | -0% | EXPMA位置 |
| 5 | sc_cci | 4.5% | +2.1% | CCI(长周期更有效) |
| 6 | ix_zs2_x_div | 2.6% | 新增 | 第二中枢×背驰交互 |
| 7 | ix_trend_x_bi | 2.6% | +1.3% | 趋势×笔交互 |
| 8 | cl_bi_dir | 2.5% | +0.7% | 笔方向(冗余) |
| 9 | sc_trend_score | 2.5% | +1.2% | 0-14评分 |
| 10 | cl_zs_min30 | 2.5% | +1.5% | 30分中枢(长周期更重要) |
| 13 | cl_zs_position | 1.3% | 新增 | 中枢连续位置 |
| 14 | cl_zs_index | 1.2% | 新增 | 中枢序号 |

### 核心发现

1. **背驰是有效的，但需要连续值** — 二分类"有/无背驰"太稀疏(罕见事件)，改成连续动量比率后立即跳升到7.2%。这说明背驰信号不应该是二元开关，而是"动量在弱化到什么程度"的连续谱。

2. **20天标签让评分特征更能发挥作用** — 短期(5天)趋势主导一切，拉长到20天后评分维度的判断力上升了6%。EXPMA位置(5.8%)和CCI(4.5%)是最有用的两个评分指标。

3. **"第二个中枢+背驰"交互有效** — 用户直觉准确：排除第一中枢(趋势刚开始)后，第二中枢的背驰信号更有预测力(2.6%)。这对应缠论经典模式"趋势第二个中枢+底背驰=一买"。

4. **中枢序号+连续位置各自贡献1.2-1.3%** — 不算强但提供了三分类之外的增量信息。连续位置分数可能因为价格在中枢内的噪声太大而信号弱。

5. **AUC从0.95降到0.86是合理的** — 20天预测本质上比5天难，0.86在金融预测中仍属良好水平。标签更难→趋势方向特征从59%降到43%→模型更依赖多元特征组合。

### 数据

- 总耗时: 28秒 (77只, czsc分析变快因为不调full_analysis)
- 109,960 有效样本 (R1=53.2%, R5=46.8%)
- 21 个特征 (缠论8 + 评分8 + 交互5)

### 待解决

- [ ] **时序分割**: 当前仍是随机8:2分割，同标的bar可能跨训练/测试集
- [ ] **30分钟中枢细化**: 目前也只有离散三分类，应该同样加序号+连续位置
- [ ] **背驰衰减**: 当前背驰分数仅赋给当前笔覆盖的bar，笔结束后归零。是否应该向前衰减？
- [ ] **信号群特征**: 仍未落地

## 实验 #3: 时序分割验证 (2026-06-02)

### 背景

实验#1/#2 使用随机8:2分割训练/测试集（sklearn train_test_split），同标的的bar可能同时在训练集和测试集中，导致AUC可能高估模型在真实交易场景中的表现。本实验改用严格时序分割验证。

### 方法

- 全部 109,960 行样本按 `_date` 字段全局排序
- 前80%时间窗口 → 训练集，后20% → 测试集
- 切分点自动落在 2024-09-12 (训练: 2015-09 ~ 2024-09, 测试: 2024-09 ~ 2026-06)
- 特征和模型完全不变，只改分割方式

### 结论

| 指标 | 随机分割 | 时序分割 | 变化 |
|:--|:--|:--|:--|
| AUC | 0.8582 | **0.8294** | **-2.9pp** |
| 准确率 | 76.9% | 75.2% | -1.7pp |
| R1精确率 | 0.78 | **0.86** ↑ | 对"涨"预测更精确 |
| R5精确率 | 0.75 | **0.59** ↓ | 对"跌"预测假阳性多 |
| 特征排名结构 | — | 完全一致 | 无变化 |

**核心发现：模型泛化能力基本确认。**

1. AUC 0.83 在时序分割下仍属良好水平（金融时序通常 0.75-0.85），不是随机分割造假。
2. **特征重要性排名完全不变**：笔方向~44%、背驰~6.5%、评分~20%。说明特征是稳定的，不依赖于切分方式。
3. **R1精确率反而升到0.86**：上涨信号的规律性在过去9年中一致。模型说"要涨"的时候比较可信。
4. **R5精确率跌到0.59**：对"跌"的预测假阳性多(1853个假跌)。这可能是因为下跌在事后看起来有规律，但事前特征不明显。历史上的大跌往往是"突然的"。
5. **切分点在2024年9月**：测试集覆盖最近9个月，跨越多只股票的完整行情阶段，验证时间够长。

### 数据

- 方法：全局按 `_date` 排序 + 前80%训练后20%测试（一行代码改动）
- 总耗时：67秒（77只标的）

## 实验 #4: 背驰衰减 + 信号群联立 + CCI背驰 + 去重 (2026-06-02)

### 背景

实验#2 暴露了三个问题：

## 实验 #5: MACD面积背驰替换笔力度背驰 (2026-06-02)

### 背景

实验#2~#4 的 `_detect_divergence_from_bi` 用的是段内首尾笔的 power_price / power_volume / length 三维力度比较。用户认为这不是真正的缠论背驰——"背驰需要有中枢的参与和中枢前后两段能量的对比，能量用MACD红绿柱面积更好"。

### 改动

从 czsc 的 `zdy_macd_bc_V230422`（zdy.py:525）移植 MACD 面积背驰：

| 改动 | 说明 |
|:--|:--|
| `_compute_macd_divergence` 替换 `_detect_divergence_from_bi` | 不再比笔的 power_price/volume/length，而是：段末取5/7/9笔 → 中间n-2笔构成中枢(比高低点重叠) → 提取进入笔 vs 离开笔的macd_hist → 累加红柱(上涨)/绿柱(下跌)面积 → 离开段面积 ≤ 进入段面积 × th% |
| 删除旧的段内笔力度比较 | 不再三维衰减比率 |
| **未保留 forward-fill 衰减** | 实验#4 的 20-bar 衰减在新版本中未实现，背驰分数仍只赋给末笔 bar |

**算法细节**（来自 `zdy_macd_bc_V230422`）：

```
1. 以段末 5/7/9 笔为候选结构
2. ZS = bis[1:-1], 有效性: zg=min(上涨笔.high) > zd=max(下跌笔.low)
3. 底背驰(下跌模式): bi1.high=max_high, bi2.low=min_low, 绿柱面积缩小
4. 顶背驰(上涨模式): bi1.low=min_low, bi2.high=max_high, 红柱面积缩小
5. th=0.5 (对应原始 th=50): 离开段面积 ≤ 进入段面积 × 50%
6. 分数 = 1 - area_ratio, 范围 0~1(底背驰) / -1~0(顶背驰)
7. 只赋给段末笔覆盖的 bar
```

## 实验 #6: 笔力度背驰 + MACD面积背驰 融合验证 (2026-06-02)

### 背景

实验#5 用 MACD 面积背驰替换了笔力度背驰（power_price/volume/length）。用户提出：两者逻辑互补，何不融合使用？—— 只有两方法方向一致时才出信号，牺牲信号数量换取单事件可靠性。

### 改动

在 [notebook/compare_divergence_methods.py](notebook/compare_divergence_methods.py) 中实现对比分析脚本，在 [notebook/explore_xgb.py](notebook/explore_xgb.py) 中新增融合函数：

| 改动 | 说明 |
|:--|:--|
| `_detect_bili_divergence(bi_list, row_dates, segments)` | 笔力度背驰：段内首尾笔比较 power_price/power_volume/length，三维衰减比率 |
| `_compute_macd_divergence(bi_list, row_dates, daily_rows, segments, th=0.5)` | MACD面积背驰：段末5/7/9笔→中枢→进入vs离开段MACD绿/红柱面积对比 |
| `_compute_combined_divergence()` | **融合函数**：两者方向相同才出信号（同号取平均），方向相反或任一为零则归零 |
| `build_dataset(div_th=0.5)` | 接受 `div_th` 参数透传给 MACD 函数 |

融合逻辑：
```python
def _compute_combined_divergence(...):
    macd_div = _compute_macd_divergence(...)
    stroke_div = _compute_stroke_divergence(...)
    combined = {}
    for i in range(n):
        m, s = macd_div.get(i, 0), stroke_div.get(i, 0)
        if m * s > 0:              # 同一方向
            combined[i] = (m + s) / 2  # 取平均
        else:
            combined[i] = 0.0
    return combined
```

## 实验 #8: 板块势能评分 — 通达信原公式 Python 精准翻译 (2026-06-03)

### 背景

目标：用一个经过通达信实盘验证的公式（RSI势能2），直接对 269 个概念板块指数（880xxx）做势能排名。排名高的板块筛选出的量领创新高个股，主题支撑更强。

### 方案演变（两个版本）

**v1（错误）** — 对成分股逐股算分再平均：
- 读取 block_gn.dat 获取板块成分股 → 逐股读 .day 文件计算势能 → 板块聚合
- 问题①：通达信公式是应用在**板块指数**上，不是成分股上。逐股算分丢失了板块整体的价格动量结构
- 问题②：`np.where(SMA!=0, arr/SMA, 0)` — SMA 前 N-1 根是 NaN，numpy 中 `NaN != 0` 返回 `True`，导致 NaN 传入除法 → EMA 全链 NaN → 短期+中期 47% 权重丢失
- v1 结果：培育钻石 26.2（vs 通达信 9.68），数值放大 ~2.7x，且排名部分扭曲

**v2（正确）** — 直接用 880xxx 板块指数日线：
- 从 tdxzs.cfg 解析板块名称↔880xxx 代码映射（category=4 概念板块 269 个）
- 读板块指数 .day 文件（如 sh880754.day → 培育钻石）
- 逐 bar 向量化计算通达信原公式，输出 X_1
- 关键修复：`_safe_div` 用 `np.divide(where=mask)` 替代 `np.where` 模式，只对 `np.isfinite(denom)` 的分母做除法

### 通达信公式逐字翻译

```
超短周期 := IF(VOL/MA(VOL,5)>1.5,3,5);  
超短强度 := ATAN((MA(C,超短周期)/REF(MA(C,超短周期),1)-1)*100)*57.3;
短期强度 := EMA(ATAN((MA(C,13)/REF(MA(C,13),1)-1)*100)*57.3, 2);    
中期强度 := EMA(ATAN((MA(C,20)/REF(MA(C,20),1)-1)*100)*57.3,3);  
长期强度 := ATAN((MA(C,60)/REF(MA(C,60),20)-1)*100)*57.3;
量能因子 := MAX(VOL/REF(VOL,1), MA(VOL,5)/REF(MA(VOL,5),1)); 
强度放大系数 := IF(量能因子>1.8,1.25, IF(量能因子>1.3,1.15,1));
势能评分 := (超短*0.45*boost + 短期*0.3 + 中期*0.2*0.85 + 长期*0.05)*100/60;
连续强势天数 := BARSLAST(势能评分<REF(势能评分,1)); 
X_1 :(势能评分 + 连续强势天数*0.015)*0.1;
```

### 实验 #3: Type A 阈值优化 + Type B 理论基础确认 (2026-06-05)

### 背景

Rolling FM + Type A/B 双成长叙事 + CAPEX 分析的独立数据层已跑通（v1版本）。但在数据验证中发现两个核心问题：
1. **Type A 原版阈值太严**（连续4季净利润增长>20%）→ 0只纯Type A，28只A+B（靠Type B补上的）
2. **Type B 包进去太多** — 银行、电力、资源股靠营收持续性混入Type B，但都不是交易选择方向

### 讨论结论

**阈值扫描覆盖**: 对5428只A股做全市场参数扫描，测试了 4/6/8季窗口 × 5%/8%/10%/12%/15%/20%均值 × 50%/75%/100%正增长比例 的全部组合。

**Type A 选定阈值: 4季均值>12% + 至少3季正增长**

理由：
- 全市场命中 1908只（35%），淘汰边缘股70只
- 量领宇宙常青树(3+月) 覆盖率 5/12（42%）— 比10%少2只（长江电力、浪潮信息），但这2只本身靠Type B营收持续性站住，不算丢失
- 漏掉的7只常青树全部净利润低增长或负增长，靠Type B留在宇宙里，Type A本来就不该抓到它们

**Type B 的理论基础确认** — 来自资深交易者十几年的经验总结：
> "营收一般都是真的，但是利润企业可以藏着。"

这句话深刻验证了Type B路径的合理性：
- **Type A依赖净利润** → 利润可以通过折旧政策、减值准备、关联交易、收入确认时点等手段调节
- **Type B依赖营收** → 营业额很难造假（有增值税发票、银行流水、客户验证）
- **结论**: Type B（营收扩张→CAPEX收获期）比Type A（净利润PEG）更可靠，尤其在A股市场

## B类突发事件检测 — 消息面关键词匹配引擎 (2026-06-05)

### 架构

```
gpcw 季报数据(Fama-MacBeth)
    →

    →
  Type A/B 成长叙事(营收增速+盈余质量)
    →
```

## 实验 #15: 信源聚合管道 + AI 写作模板系统 (2026-06-07)

### 背景

六层宏观数据框架（中国宏观/日本套息/全球流动性/消息面/US市场/US宏观）全部跑通后，管理层面上出现新问题：**run_daily.bat 越来越臃肿**，信号计算+信源拉取混在一起，一次运行半小时，且信源数据（公众号/消息面/US市场）和信号计算（通达信/周期分析/回测）没有独立的运行节奏。

同时，实验#13（Dorian框架对标洪灏）发现输出"中规中矩"——有数据没观点、有结论没预测。需要一个**独立的叙事层**，把六层数据的结论用因果链串起来，而不是平铺数据。

### 设计方案

**核心分离**: 把"拉外部数据"从 run_daily.bat 完全拆出来，形成两个独立 pipeline：

```
run_daily.bat      ← 纯计算：通达信同步 → 信号 → 周期分析 → 回测 → 报告
update_sources.bat ← 纯信源：公众号 → 消息面 → 流动性 → US宏观 → 日本 → US市场 → 基本面 → AI分析
```

**update_sources.bat** (9步):
| 步 | 模块 | 数据源 |
|:--:|------|--------|
| 1 | 微信公众号 | MPTEXT API → wechat_articles/ |
| 2 | 消息面冲击 | 鼓掌WebSocket+见闻+东财 → sentiment_shock.json |
| 3 | 流动性 | BTC/VIX/DXY/M2/信用脉冲 → liquidity_monitor.json |
| 4 | US宏观敏感度 | Fed/CPI/ISM/非农 → us_macro_sensitivity.json |
| 5 | 日本宏观+套息 | BOJ/FXY/CPI → japan_macro.json |
| 6 | US ETF动量 | 52只ETF RSI势能2 → us_sector_momentum.json |
| 7 | US明星股动量 | 64只 RSI势能2 → us_star_momentum.json |
| 8 | 概念链轮动 | 30条链 → us_concept_momentum.json |
| 9 | 基本面因子 | gpcw → fundamental_profile.json |
| → | AI分析 | gen_source_summary.py --ai → reports/sources/ |

每步独立错误处理（`set /a ERR_CNT+=1`），不阻断后续步骤。

**gen_source_summary.py** — 信源聚合摘要生成器:
- 读取全部7个JSON + wechat_articles 目录
- 无AI模式：纯数据摘要 → `reports/sources/YYYYMMDD_sources.md`
- `--ai` 模式：加载 `prompts/source_analysis_prompt.md` 作为写作模板 → 调用 NVIDIA V4 Flash API → 追加AI分析到摘要
- 安全设计：`_read_json()` 返回 None，`_clean()` 处理编码乱码
- 关键修复：Windows GBK 编码问题，所有文件读入指定 `encoding="utf-8"`

**prompts/source_analysis_prompt.md** — 可独立编辑的写作模板（不碰代码即可改风格）:

五段结构：
1. **锚点事件开篇** (1段) — 选当日最重要1个具体事件，有时间/数字/对比
2. **因果链展开** (2-3段) — A→B→C完整传导，每层有数据支撑
3. **反直觉发现** (1-2段) — "表面上是X，实际上是Y"的矛盾点，全文洞察高峰
4. **多源交叉验证** (1段) — 同一判断有2-3个独立信源指向
5. **可操作结论** (1段) — 1-2个具体可验证判断+时间窗口

逻辑规则：
- 每段必须回答三个问题之一：为什么重要？(因果)、和什么有关？(关联)、别人没看到什么？(反直觉)
- 禁止：平铺数据("流动性0.113")、分块报告("公众号部分...")、正确废话("需警惕风险")
- 语气：交易员语言，短句，一段不超过5行，800-1200字，可以打比方

数据源融合顺序（按决策权重）：
1. 宏观锚定：流动性+US宏观+日本 → 定方向
2. 消息面发酵：sentiment_shock → 定当前主要矛盾
3. 公众号观点：猫笔刀/表舅/卓哥/亨特/laoduo → 交叉验证
4. 市场结构：US ETF/明星股/概念链动量 → 定资金流向
5. 基本面：因子溢价 → 长期验证

每次必须覆盖的5个审视角度：
1. 全球流动性松紧？（流动性因子+日本套息+美元方向）

## 实验 #16: 分析框架定位修正 + 六步流程强制 + P0/P1/P2 数据补齐 (2026-06-07)

### 背景

会话压缩重启后，发现两个核心问题：
1. 我之前错误地把"公众号作者"当作分析主体——公众号应该是交叉验证，不是分析替代
2. 分析框架停留在prompt文件层面，没有成为行为习惯——用户担心"只停留在prompt上，没有实际应用"

### 修正一：分析框架正确定位

**AI框架做主体推理，公众号是交叉验证（最低优先级）**：
```
① 宏观锚定 → 我自己判断regime
② 市场结构 → 我自己看资金流向
③ 消息面触发(shock_detector) → 知道出了什么事
④ 链式推理 → 我自己连接因果链
⑤ 公众号观点 → 最后一步交叉验证
```

### 修正二：shock_detector 定位

- **之前**：完整impact评分+多级严重性
- **现在**：只做"出了什么事"的触发层
- 触发后的分析推理走AI六步框架，不由关键词匹配决定

### 修正三：Dorian 32篇文章完整研读

通读灰岩金融科技全部32篇文章（2026-03-18~2026-04-19），提炼出完整体系：

**底层哲学**：
- 交易 = Decision Structure Game，不是信息游戏
- Edge = Structure × Timing × Risk Constraint，不是 Information
- 专业vs散户分水岭：散户定义止盈，专业定义**失效条件**

**六步分析流程（本次核心产出，以后所有分析强制走完）**：
```
1. 变量拆解（洪灏：找到最关键1-2个变量，不是罗列）
2. 传导链（A→B→C，每层拷问"所以呢"）
3. 历史对标（像在哪+不一样在哪）
4. 情景推演（基准/乐观/悲观，互斥穷举）
5. 交易映射（进攻/防守？方向倾向？）
6. 失效条件 ← Dorian：最重要的一步（没有失效条件的分析=观点，不是交易判断）
```

**失效条件四种类型（每个分析至少写2条）**：
- ① 数据失效：CPI突破X阈值
- ② 逻辑失效：传导链中A→B不成立
- ③ 时间失效：X天内没发生Y
- ④ 结构失效：市场定价结构变了

**Dorian认知框架核心概念**：
- 美联储 = 巨船驾驶舱（罗盘/油量/扩音器/航海日志），当前是"Waiting Cycle"

- 叙事驱动论：数据本身不移动市场，对数据的解释才移动市场
- TACO + Gamma：方向对了还亏钱是因为市场结构不允许你的判断被兑现
- AI = 认知外骨骼（不是答案机器），10条可直接用的投研Prompt
- 流程 vs 系统：流程=手动步骤，系统=流程+记忆层+触发层+执行层

## 实验 #17: 节点地图系统 — 板块行情区间×龙头梯队识别 (2026-06-08)

### 背景

用户发现同花顺有一个"板块节点"功能：K线图上标注每波行情区间、龙头股(龙1/龙2)、涨幅、参与广度。用户认为这是验证"★买信号是否有板块效应支撑"的小突破口。

设计文档：[docs/node-map-design.md](../docs/node-map-design.md)（v0.3对齐同花顺输出格式）

### 实现

**`tools/node_map.py`** (~700行)，纯TDX数据，无外部依赖。

**数据来源**：
- 板块指数日线: `sh/lday/sh880xxx.day` (654个文件)
- 板块名称→880xxx: `tdxzs.cfg` (category=4=概念板块, 270个)
- 个股→板块映射: `block_gn.dat` (通过BlockReader, 40708行, 5343只个股→270板块)
- 个股名称: `stock_names.csv` (47058条, 8位代码→6位剥离)

**波动检测逻辑**：
```python
# SMA5平滑 → 局部极值(order=8) → valley→peak→valley = 一波行情
# 过滤: gain>=8%, min_days>=3, 合并重叠浪
```

**龙头识别**：
```python
# 在浪窗口内,计算每只成分股的峰值涨幅
# 按peak_gain排序 → 龙1/龙2/龙3/龙4/龙5 (取positive_returns)
# 同时返回all_returns(含跌的)用于参与广度统计
```

**节点质量评分 (0-12分 → A/B/C/D四级)**：
| 维度 | 分值 | 逻辑 |
|------|------|------|
| 参与广度 | 0-3 | positive_rate >= 60→3, >=40→2, >=25→1 |
| 参与深度 | 0-3 | strong_count(涨超10%) >= 20→3, >=10→2, >=5→1 |
| 龙头梯 队 | 0-2 | 龙1/龙3 >= 1.8×→2, >= 1.3×→1 |
| 持续时间 | 0-2 | duration >= 30d→2, >= 10d→1 |
| 中位涨幅 | 0-2 | median >= 20%→2, >= 10%→1 |

A级(>=9): 确认真节点。B级(6-8): 大概率真节点。C级(3-5): 存疑。D级(<3): 噪音。

**节点上下文标注**（全部从TDX计算，不依赖外部API）：
- 大盘环境（上证EXPMA趋势+窗口内涨跌幅+量能趋势）
- 板块位置（底部反转/高位加速/趋势延续 + 相对大盘alpha）
- 流动性代理（上证成交额窗口前后对比）

### 关键bug

1. **参与率永远是100%**: `find_leaders` 只返回positive_returns → `analyze_wave_breadth`只能看到涨的股票 → 修复: 返回all_returns(含跌的)，参与广度才有意义
2. **上证指数市场检测错误**: "000001" startswith "00" → 判为sz → 修复: 000xxx → sh
3. **stock_names.csv 8位代码匹配失败**: 缓存存的是"sz000001"，filter用len(code)==6过滤 → 修复: 统一剥离sh/sz前缀

### 结果

- 270个概念板块 → 3750个行情节点（210秒全量扫描）
- 等级分布: A=1972, B=1096, C=310, D=372
- A+B=3068个确认节点（81.8%），覆盖233个板块
- D级多为无成分股数据的新板块（如DeepSeek概念）
- 验证案例: 光伏概念2019-01-02至2019-04-18，+37.7%，龙1+222.5%，143/155只涨超10%，这是2018年底后的光伏大反弹

## 实验 #19: ★买→节点映射→贝叶斯收缩 (2026-06-08)

### 背景

用户在三阶段规划中明确：Phase 3 = ★买信号映射到已确认节点 → 按节点类型分组 → 贝叶斯收缩修正置信度。用户偏好贝叶斯方案（而非简单乘法），原话："我好像偏好上就喜欢方案B 还是带了贝叶斯相关的逻辑好像"。

### 贝叶斯收缩公式

```
P_adj = (N_global × P_global + N_group × P_group) / (N_global + N_group)
```

- N_global=200: 全局先验有效样本权重 — 越大越不相信小组数据
- P_global=48.3%: 全量回测金叉策略胜率(from backtest_signals.py)
- P_group: 该分组的代理胜率（基于节点等级+宏观环境加减）
- 核心效果: 小组样本量小时自动向全局均值回拉，避免过度信任稀疏数据

### 实现

**`tools/star_buy_node_map.py`** (~330行)

**个股→板块反向索引**: `build_stock_to_sectors()` — 从block_gn.dat构建5343只个股→板块映射

**节点匹配**: `match_signal_to_nodes(signal_date, stock_code, ...)` — 对每个★买信号，找到其所属板块中哪些A/B节点窗口覆盖该日期。窗口格式修复后正常工作。

**分组维度**: 节点等级(A/B) × 中国宏观(宽松/中性/收紧) × 流动性(宽松/中性/收紧) → 8个有效分组

**代理胜率**: 无法从回测直接提取每个信号的PnL，用节点质量做合理代理：
- A级+宽松宏观 → +15%(0.483+0.10+0.05=0.633)
- A级+中性 → +10%
- B级+中性 → +5%
- 收紧宏观 → -5%

### 结果

- 345个历史★买信号（来自14只跟踪标的），146个(42.3%)落在A/B节点窗口内
- 分组贝叶斯收缩后:

| 分组 | 信号数 | P_group | P_adj | 含义 |
|------|--------|---------|-------|------|
| A级+中性+中性 | 37 | 58.3% | 49.9% | 有板块效应支撑，+1.6% |
| A级+中性+宽松 | 35 | 63.3% | 50.5% | 最佳组合，+2.2% |
| A级+中性+收紧 | 30 | 58.3% | 49.6% | 板块好但宏观紧，+1.3% |
| B级+中性+中性 | 11 | 58.3% | 48.8% | B级效应弱，+0.5% |

- 信号在节点早期(days_from_start < 30%)→额外+5%位置加成
- 信号在节点晚期(>70%)→-3%追高风险折扣
- 不在任何节点内的信号(57.7%)→不调整，使用全局先验48.3%

### 局限性

1. **P_group是代理而非真实胜率** — 需要完整买卖闭环才能得到每组的真实PnL。当前用节点等级+宏观环境的固定加减做代理，方向正确但精度有限
2. **N_global=200的合理性** — 主观选择，需后续交叉验证确认最优值
3. **仅覆盖金叉策略** — ma_delayed策略(34.7%胜率)的贝叶斯分组未展开

### 关键词

#贝叶斯收缩 #★买映射 #节点分组 #置信度修正 #全局先验 #P_adj #代理胜率 #板块效应支撑

### 一、HALO重定义：从"通胀框架"到"AI焦虑→避险资产"

**问题**：最初将HALO理解为"通胀→高利润资产重估"的化工子框架，完全偏离用户本意。

**正确含义**：HALO = Heavy Assets + Low Obsolescence。核心问题不是"什么资产在通胀中受益"，而是**"AI颠覆一切的时代，什么资产最安全"**：

```

  → 软件/互联网/信息资产"护城河"被AI侵蚀
    → 数据壁垒不再有效（所有人都能用AI获取和处理数据）
      → 资金寻找技术迭代慢、AI颠覆不了的实物资产
        → 重资产（建厂300亿，AI不能帮你少花一分钱，资本开支是真实壁垒）
          → 低淘汰率（技术路线稳定，不存在被颠覆风险）
            → AI意外创造额外需求（算力+电力+冷却+MLCC+面板）
```

### 三、传导链工作台：地缘→宏观→产业链的"非标准推导"

**问题**：常规研报覆盖"标准逻辑"（公司基本面→估值），但**跨环节的传导链**（地缘事件→宏观变量→产业链）难以被常规研报覆盖，却是实盘价值最高的部分。

**方案**：在 `narratives/overseas/transmission_chains.md` 建立"传导链工作台"，每条链包含：
- **起点**：触发事件（地缘/政策/自然/价格信号）
- **传导路径**：A→B→C→D→E 完整逻辑链，每层标注可观测指标
- **关键中间变量**：必须盯住的信号（油价/运价/库存等）
- **终点**：具体影响的产业链/品种/标的
- **验证条件**：什么信号说明传导在生效
- **失效条件**：什么信号说明链条断了
- **信源**：验证过这条链的报告/数据

### 五、方法论反思：框架建设的诱惑 vs 精读数据的价值

用户反复批评的核心问题（记录为长期教训）：

1. **已有数据不读就建新框架**：2000+篇研报/文章已经包含了大量数据，正确的顺序是先精读、提炼逻辑链、再用传导链模型去组织，而不是反过来先建框架再往里填数据
2. **"海外变量观察站"擅自创建**：用户明确否决 `narratives/overseas/variables.md`，指出"真的有用么？你自己会扫描会看么？"——变量跟踪站如果没有日常维护机制就是空架子
3. **标题命中搜索盲区**：关键数据（WF6涨价）存在于一篇标题不包含关键词的文章中，纯标题命中搜索会漏掉大量有价值的内容
4. **聚合消息的"信号类型"分类缺失**：当前只按事件类型分类（关税/制裁/地缘），缺乏按"信号类型"（涨价/供给收缩/流动性冲击/政策转向）的分类维度

**正确的三件事**（来自用户的明确方向）：
1. **精读研报** — 不是泛读，是逐篇提炼逻辑链和结论，按产业链方向分配
2. **精读聚合消息** — 从每日聚合消息中提取涨价/供给/政策信号，映射到产业链
3. **逻辑链补全** — 发现的新逻辑链写入传导链工作台，再分发回各叙事线和叙事看板

## 实验 #23: 知识图谱补全 + 信号事件流提取器 (2026-06-10)

### 背景

两个关联任务：(1) 产业链知识图谱（industry_kg.json）从 CKG 67K 产品中提取后，存在大量产品缺口——关键功率半导体（SiC/GaN/IGBT）、半导体设备零部件、先进封装（HBM/Chiplet）、AI算力链产品等频繁出现在研报/新闻中但未被建模；(2) 专家系统架构需要机器可消费的结构化信号输入，替代现有的纯人类日报形态——从每日数据源中提取离散信号事件，用 KG 映射到叙事链，输出 JSON 事件流。

### 一、知识图谱补全 (supplement_kg.py)

**问题**：`build_knowledge_graph.py` 以申万行业分类从 CKG 提取产品→链映射，但大量"研报高频词"（SiC、HBM、光刻机等）未被覆盖，导致后续信号提取的 recall 严重不足。

**补全方案**：人工 + 网络搜索 + 研报交叉验证，补充 120 个产品，覆盖 15 个方向：

| 方向 | 补充数 | 代表产品 |
|------|--------|---------|
| 功率半导体 | 16 | SiC衬底/SiC MOSFET/GaN/IGBT/超级结MOSFET |
| 半导体设备 | 15 | 光刻机/刻蚀/CVD/PVD/ALD/CMP/清洗/离子注入 |
| IC设计 | 8 | MCU/GPU/AI加速卡/FPGA/DSP/RISC-V |
| 半导体材料 | 14 | 大硅片/ArF光刻胶/CMP抛光液/电子特气/靶材 |
| 先进封装 | 12 | Chiplet/HBM/CoWoS/2.5D/3D/Fan-Out/SiP |
| 半导体配套 | 5 | 洁净室工程/高纯管路/FMCS |
| PCB/被动元件 | 6 | IC载板/ABF/HDI/MLCC |
| AI基础设施 | 6 | AI服务器/液冷/光模块/800G/硅光 |
| 新能源 | 8 | 钙钛矿/HJT/TOPCon/固态电池/钠电/磷酸铁锂 |
| 其他 | 30 | 创新药(GLP-1/ADC/CAR-T)/新材料/低空经济/军工 |

**KG 匹配修复**：申万行业代码匹配从原始 2-digit 前缀改为"4-digit 宽匹配过滤 + 6-digit 精确匹配分配"混合策略，解决了之前 333 个产品被错误分配到同一链的问题。

**最终状态**：3,829 产品节点（CKG 3,709 + 补充 120），50 条链全覆盖，product_graph 1.0 MB。

### 二、Layer 3 概念验证 (kg_demo.py)

在 `tools/kg_demo.py` 中验证了产品→链子字符串映射方案：
- 遍历 product_graph 所有产品名，对输入文本做大小写不敏感子字符串匹配
- 跳过短名称（<2字符）和噪声前缀（`%`/`)`/`-`/`#`/`'`）
- 每个匹配产品为其所属 chains 投票，取 top N
- 4 个测试案例验证（半导体限制/AI算力/新能源车/国产替代）
- 验证通过后删除 demo 文件，逻辑合并到 signal_extractor.py

### 三、信号事件流提取器 (signal_extractor.py)

**架构**：~730 行，4 个核心组件

| 组件 | 职责 |
|------|------|
| DataSourceReader | 读取 9 个 JSON 数据源 + 微信公众号 .txt 全文 |
| KGMatcher | 通过 product_graph 做文本→叙事链映射 |
| SignalExtractor | 10 个源独立提取方法，每源生产离散信号事件 |
| FreshnessTracker | MD5 哈希去重，7 天过期 |

**10 个数据源**：

| # | 源 | 信号类型 | 映射方式 |
|---|------|---------|---------|
| 1 | sentiment_shock.json | 突发事件 | SHOCK_TYPE_CHAINS 预定义映射 + KG 文本补充 |
| 2 | liquidity_monitor.json | 流动性状态/因子方向 | HIGH_BETA_CHAINS 等预定义 + 因子级映射 |
| 3 | macro_snapshot.json | 中国宏观环境 | 宏观标签直接输出 |
| 4 | us_macro_sensitivity.json | US 宏观/因子 | factor_chain_map 预定义 |
| 5 | japan_macro.json | 日本套息 regime | regime→direction 映射表 |
| 6 | us_sector_momentum.json | US ETF 类别动量 | ETF_CATEGORY_CHAINS 映射 |
| 7 | us_star_momentum.json | US 明星股类别动量 | STAR_CATEGORY_CHAINS 映射 |
| 8 | us_concept_momentum.json | US 概念链轮动 | CONCEPT_CHAIN_MAP 映射 |
| 9 | fundamental_profile.json | 基本面因子方向 | 因子名称关键词匹配 |
| 10 | wechat_articles/*.txt | 公众号正文 | KG 全文子字符串匹配 |

**关键设计决策**：
1. **不设独立的 Layer 2 信号类型分类层**——类型判断嵌入各源提取器的直接逻辑中（direction/impact 直接来自数据源字段），避免额外抽象
2. **公众号读全文不读标题**——.txt 文件含完整正文，跳过标题/时间/链接/分割线四行头，用正文做 KG 匹配，recall 大幅提升
3. **双重 KG 映射**：结构化源走 `direct_code`（预定义映射表）+ `kg_text_match`（文本匹配补充），非结构化源（公众号）纯走 `kg_text_match`
4. **7 天去重缓存**：_freshness_cache.json，MD5(raw_text[:60]) 做 key

**输出**：
- JSON: `signals/tracking/_signals/daily_signals/YYYYMMDD_signals.json`（~44KB，62-81 条信号）
- Markdown: 同上目录 `YYYYMMDD_signals.md`（链汇总条形图 + 源明细）
- 每日管线：`update_sources.bat` 步骤 [11/11]

**实测结果**（2026-06-10 数据）：

| 指标 | 值 |
|------|-----|
| 总信号数 | 81 条 |
| 数据源加载 | 10/10 |
| 激活链数 | 33 条 |
| 公众号信号 | 22 条（来自 8 个信源正文） |
| 主导方向 | negative（当日情绪偏空） |
| 最强链 | narrative#11 AI基础设施（15 条信号） |

### 五、与实验 #22 的关联

实验 #22 指出的"聚合消息的信号类型分类缺失"问题，在 signal_extractor.py 中得到部分解决——不再按"事件类型"分类，而是每源按自身数据结构提取 direction/impact/chains。但 `shock_detector.py` 本身的分类改造（涨价/供给收缩/流动性冲击）尚未涉及，等待用户示范后再动手（待办A）。

### 短板

1. **KG 匹配仅子字符串**——无分词/同义词/语义嵌入，"SiC"和"碳化硅"匹配不同产品且不互通
2. **US→A 股映射弱**——us_macro_sensitivity.json 的 560 个 A 股 sector 尚未有效映射到 50 条叙事链
3. **公众号仅取前 3 篇**——部分信源（如 laoduo）每日多篇，可能遗漏
4. **无增量模式**——当前全量重跑，未利用 freshness 做增量追加

## 实验 #24: 全量研报并行精读 → 叙事系统全面升级 (2026-06-10)

### 背景

叙事系统（53条产业链时间线 + S/A/B/C/D分级 + 海外传导链）初版于6月9日建立，基于500字模板通读。但中信建投证券研究公众号有5285篇历史文章（其中1273篇来自2026年1-6月），蕴藏大量未利用的研报数据——包括行业深度、中期策略、月度跟踪、点评快报等。需要一次全面的"研报灌溉"来提升叙事系统的数据密度和时效性。

### 一、并行 Agent 架构验证（里程碑）

**问题**：单 agent 逐个读1273篇研报约需数小时，无法在会话窗口内完成。

**方案**：利用 DeepSeek V4 Flash 的1M上下文窗口 + Claude Code 的并行 Agent 工具，12个 agent 同时运行，每 agent 负责 ~100篇研报。

**Phase 1 实测数据**：

| 指标 | 值 |
|------|-----|
| 总研报数 | 1273篇（2026-01至06）|
| 有效研报 | ~1100篇（剔除152篇每日市场点评噪声）|
| 并行 agent 数 | 12（首次尝试，全量覆盖）|
| 实际精读 | ~350篇（每 agent 25-30篇）|
| 运行时间 | ~4分钟 |
| 窗口压力 | 71%→ 结果正常返回 |
| 阻塞数 | 0 |

**结论**：并行 agent 架构在量化系统中验证可行，12路并发无阻塞。

### 二、跨块分析引擎 (signal_deep_reader.py)

`tools/signal_deep_reader.py`（632行），新增两个核心组件：

**Layer 2 → Layer 1 反哺机制**：从聚合日报的10个数据块中读取原始数据，用 LLM 识别跨块关系，产出结构化信号。

**4种信号类型**（定义在 `prompts/cross_block_analysis_prompt.md`）：

| 信号类型 | 含义 | 优先级 |
|---------|------|--------|
| 块间冲突 | 不同数据块指向相反方向 | 高（提示市场分歧）|
| 块间共振 | 多个数据块指向同方向 | 高（趋势强化确认）|
| 单块确认 | 单一块内出现明确信号 | 中 |
| 全局预警 | 系统级风险信号 | 最高 |

**6月10日实测**：6条跨块信号，包括"结构性利空而非系统性危机"（冲突）、"中期风险预警"（共振）、"地缘→系统性流动性危机"（全局预警）。详见 [20260610_aggregation_signals.json](signals/tracking/_signals/daily_signals/20260610_aggregation_signals.json)。

**两种模式**：
- `--aggregation-only`：仅聚合分析（快速，~30秒）
- 全模式（无参数）：聚合分析 + 逐篇精读（较慢，未完成）

### 六、方法论反思

1. **并行 agent 的性价比**：12路并行4分钟 = 48 agent·分钟，相当于单 agent 连续工作约1小时的工作量。窗口压力从71%→满，但结果成功返回。阈值：建议不超过12个并行 agent，且总研报数控制在1500以内。

2. **全量灌溉 vs 增量更新**：本轮是首次"全量灌溉"，后续应切换到增量模式——每篇新研报入库时自动触发单 agent 分析，只更新受影响的链，避免重复全量。

3. **研报分级**：1273篇中约12%（152篇）为每日市场点评（"今日A股调整"等），叙事价值极低。后续可建立"研报质量分级"：深度报告(★★★) / 行业跟踪(★★) / 每日点评(★)，只对★★以上做叙事更新。

4. **叙事分级层需要定期维护**：S/A/B/C/D 不是静态标签，建议每2周检查一次重大事件驱动的重新评级，每季度全量重评。

### 短板

1. **传输链编辑中途失败**——Chain #006 的初次 Edit 因 tab/space 缩进不匹配失败，需人工重试（已修复）
2. **未验证22条时间线的每处改动**——agent 输出在上下文但未逐条核对，可能存在幻觉
3. **深度报告中定量数据未被完整提取**——agent 偏向提取定性结论，忽略表格/数字细节，后续需补充定量提取层
4. **跨块分析全模式未完成**——`--aggregation-only`可用，但全模式（逐篇精读）因 LLM 调用过多挂了
