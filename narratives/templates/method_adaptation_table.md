# 叙事时间线方法适配表

基于coverage_scan_v2数据（50条链 × 2299篇研报），按**标题命中数**和**叙事结构特征**分类。

---

## 一、两种方法的本质区别

| 维度 | 中央报告法（功率半导体原型） | 碎片拼凑法（先进封装原型） |
|------|---------------------------|-------------------------|
| 触发条件 | 存在1+N篇深度专门报告 | 无专门报告，信息分散在N篇其他主题文章中 |
| 时间锚定 | 精确到天（报告发布日期=拐点） | 模糊到月~周（子主题涌现窗口） |
| 主干结构 | 单条主线：旧叙事→引爆→扩散→验证 | 多条子线：不同子主题在不同时间涌现叠加 |
| 分析师归属 | 单一团队（电子/通信/电新） | 跨2-3个团队，视角不同 |
| 投入产出 | ~1小时/条，产出完整 | ~3小时/条，产出70-80%完整性 |
| 最佳场景 | 独立报告多、叙事有明确引爆点的链 | 体量小标的少、但词频高的链 |

## 四、精读时间线 vs 500字小模板的区别

你问到重点了。直接对比两个文件：

| 维度 | 现有小模板 (p0_06_power_semi.md) | 精读时间线 (narrative_timeline_power_semi.md) |
|------|----------------------------------|----------------------------------------------|
| **本质** | 静态逻辑快照 | 动态演化记录 |
| **篇幅** | ~500字，7个固定字段 | ~2500字，多阶段叙事 |
| **时间感** | 告诉你"现在逻辑是什么" | 告诉你"逻辑是怎么变成现在这样的" |
| **精确度** | "AI数据中心驱动"（笼统） | "2026年1月15日AIDC电源革命发布，叙事正式切换"（精确到天） |
| **阶段划分** | 无（单状态描述） | 4-5个阶段（旧叙事→认知铺路→范式切换→扩散→当前） |
| **关键变量** | 列出但不追踪 | 列出+标注验证状态（✅已验证/⏳待验证） |
| **节点对照** | 无 | 有（每个阶段对应节点地图的波段，验证叙事与价格共振） |
| **信息来源** | 笼统引用 | 精确到具体报告标题+日期+分析师 |
| **案例价值** | 初次了解产业链的逻辑框架 | 复盘叙事如何演化、识别当前处于哪个阶段、预判下一个拐点 |
| **更新成本** | 低（改几个字段） | 高（需重新读报告确认新阶段） |
| **使用场景** | 日常参考/快速查阅 | 深度复盘/拐点识别/投资决策时的叙事定位 |

**一句话总结**：小模板是"产业链名片"，精读时间线是"产业链传记"。名片告诉你他现在是谁，传记告诉你他经历了什么才变成今天这样。
